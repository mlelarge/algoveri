Your task is to implement the splay operation for a splay tree (a self-adjusting Binary Search Tree) and verify its correctness in ACSL using Frama-C/WP. Splaying moves a target value `v` (or the search-terminating node, if `v` is absent) to the root through a sequence of structured rotations: the Zig case (parent is root, single rotation), the Zig-Zig case (`x` and parent are both left or both right children, two same-direction rotations), and the Zig-Zag case (offset directions, two opposite rotations). Signature: `SplayNode *splay(SplayNode *t, uint64_t v)` where `SplayNode { uint64_t val; SplayNode *left, *right; }`. Preconditions: `\valid(t)` and `is_bst(t)`. The contract specifies `assigns \nothing`, `\valid(\result)`, `is_bst(\result)`, set preservation (`\forall integer x; in_view(\result, x) <==> in_view{Pre}(t, x)`), root-targeting if found (`in_view{Pre}(t, v) ==> \result->val == v`), and the fallback (`!in_view{Pre}(t, v) ==> in_view(\result, \result->val)`, i.e. the new root is some element of the tree, namely the search-terminating node — predecessor or successor). The preamble defines `in_view(t, x)` as recursive membership and `is_bst(t)` as the standard BST invariant. Verification challenges: (1) BST preservation under rotation — single, double, and triple rotations each rearrange three or four pointers; you must prove that after each rotation the recursive `is_bst` predicate still holds. This requires lemma functions about how membership distributes across rotations. (2) Set preservation — `in_view` is preserved by every rotation because rotations only re-shape pointers without adding or removing nodes; an inductive argument on tree height suffices. (3) Termination — `splay` is recursive over depth; the natural measure is the depth of `v` in the tree (or the depth at which the search terminates), which strictly decreases per Zig-Zig/Zig-Zag step (by 2) and Zig step (by 1). (4) Memory model and `assigns \nothing` — this is the subtle issue: a faithful in-place pointer-rewiring implementation will write to `t->left`, `t->right`, etc., which violates `assigns \nothing`. ACSL-specific notes: (a) The `assigns \nothing` contract is likely unsatisfiable for any in-place pointer-mutating implementation; it effectively forces a path-copying (functional) implementation that allocates new nodes for every node on the splay path and reuses pointers to untouched subtrees, mirroring the Verus signature `splay(tree: Box<Node>, v: u64) -> Box<Node>` which moves ownership. If you adopt path-copying you will additionally need `allocates ...` clauses and the contract becomes provable. This is noted as a contract limitation rather than fixed in the spec, because changing `assigns` would alter the intended semantics. (b) Use default `-wp-model Typed`; `is_bst` and `in_view` are read-only structurally-well-founded fixpoints. (c) No top-level lemmas may be added, but lemma functions for rotation invariance of `is_bst` and `in_view` are essentially required. (d) The recursion needs a `loop variant`/measure for termination (depth of `v`).
