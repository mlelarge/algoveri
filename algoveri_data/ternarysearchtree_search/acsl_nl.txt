Your task is to implement the Search operation for a Ternary Search Tree (TST) and verify its functional correctness in ACSL using Frama-C/WP. The TST node is `typedef struct TstNode { unsigned char val; bool is_end; struct TstNode *left, *mid, *right; } TstNode`, mirroring the Verus `Node { val: u8, is_end, left/mid/right: Option<Box<Node>> }`. A TST stores byte sequences with three-way branching: `left`/`right` form a BST on alternative characters at the current position, while `mid` is the trie link to the next character. Requirements: Implement `bool tst_search(const TstNode *t, const unsigned char *key, size_t len)` returning true iff key[0..len-1] is in the tree's abstract view. (1) Algorithm: if t == NULL or len == 0 return false; otherwise compare key[0] to t->val: smaller -> descend into left (key unchanged); larger -> descend into right (key unchanged); equal and len == 1 -> return t->is_end; equal and len > 1 -> descend into mid with key+1, len-1. (2) Functional Correctness: the result must satisfy `\result <==> opt_contains(t, key, len)`, where `opt_contains == (t != NULL && tst_contains(t, key, len))` and `tst_contains` recursively encodes the three-way branch from the preamble. Verification Challenges: (1) Mixed BST + Trie recursion: the predicate `tst_contains` has three disjuncts (left-branch, right-branch, equal-branch), and the equal disjunct splits on len == 1 vs len > 1. WP must unfold the predicate at each loop step matching the implementation's branch. (2) BST invariant on left/right: well_formed states all keys in the left subtree start with a byte strictly less than t->val (analogously for right). The implementation does not maintain this — it is a precondition — but the search proof relies on it: when key[0] != t->val, contains in the wrong subtree is automatically false. (3) Termination: a 2-D variant — either len decreases (mid descent) or the depth of the current subtree decreases (left/right). The simplest is a structural decrease via `\at(node, ...)` shrinking on each step. ACSL-specific notes: (a) The function is pure: `assigns \nothing;`. (b) `well_formed` is recursively defined and the BST invariant uses the higher-order quantifier `\forall unsigned char *s, integer sl; tst_contains(left, s, sl) ==> sl > 0 && s[0] < t->val`. WP can ingest this provided the loop invariant quotes the analogous "any key reachable from cur, prefixed by the consumed bytes, equals the input key" condition. (c) Use `-wp-model Typed`; pointer-only reads, no extra `\separated` clauses needed. (d) Termination: `loop variant <structural-measure>;` with a ghost depth bound. (e) No top-level lemmas needed; the recursive predicates in the preamble are sufficient.
