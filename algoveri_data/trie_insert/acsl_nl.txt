Your task is to implement the Insert operation for a Byte-based Trie (Prefix Tree) and verify its functional correctness in ACSL using Frama-C/WP. The trie node is `typedef struct TrieNode { bool is_end; struct TrieNode *children[256]; } TrieNode`, matching the Verus `Node { is_end, children: Vec<Option<Box<Node>>> }` with children of length exactly 256. Requirements: Implement `TrieNode *trie_insert(TrieNode *t, const unsigned char *key, size_t len)`. (1) Algorithm: walk the existing tree along the bytes of key, allocating a fresh node (with all 256 children NULL and is_end == false) for any missing edge. After the final byte, set `is_end := true` on the destination node. If t == NULL, allocate a fresh root first. (2) Functional Correctness: the returned pointer is non-null, satisfies `well_formed`, is not `trie_is_empty`, and satisfies `trie_contains(\result, key, len)`. Frame: every other key has identical membership before and after, expressed via the postcondition `\forall unsigned char *k, integer kl; !seq_eq(k, kl, key, len) ==> (trie_contains(\result, k, kl) <==> opt_contains{Pre}(t, k, kl))`. Verification Challenges: (1) Heap allocation: the contract uses `allocates \result;` to declare that the return value points to a fresh region. Frama-C/WP's allocation model has limited support for `\fresh` and freshly-allocated arrays; reasoning that the new children array of 256 NULL pointers is disjoint from all existing trie nodes typically requires an explicit `\separated` clause for each malloc, plus a ghost set tracking allocated nodes. This is the dominant verification difficulty for the trie family — Verus handles it for free via Box<Node> ownership. (2) Recursive structural induction: well_formed is defined recursively, and inserting a node requires re-establishing well_formed on every ancestor along the inserted path. A natural proof structure is a ghost recursive helper or a loop invariant carrying "the prefix walked so far is well_formed and the suffix is the recursively-defined insert image". (3) Frame property: proving non-touched keys keep the same membership requires careful pointer-disjointness reasoning, which again hits the `\fresh`/allocation gap. ACSL-specific notes: (a) `assigns \nothing;` is correct because pre-existing memory is unchanged; new memory is captured by `allocates \result;`. (b) Use `-wp-model Typed` (or `Typed+ref` if you need stronger framing); the malloc returns must be cast to TrieNode* explicitly. (c) Recursive predicates trie_contains / well_formed must be preserved across allocation — a `lemma` may be needed stating "fresh allocation does not touch existing well_formed subtrees", though in practice this is hard to discharge with WP's default allocation model and is the principal challenge to flag. (d) Termination: the byte index 0 <= i < len is the variant; supply `loop variant len - i;`. No top-level lemmas should be added beyond the preamble; document the allocation-model limitation as the single major obstacle.
