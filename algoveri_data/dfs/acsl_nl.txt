Your task is to implement a Depth-First-Search reachability test on a directed graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is given as a CSR-encoded `Graph` struct with `n` vertices, a `row_ptr` array of length `n + 1`, and a `col_idx` array of length `row_ptr[n]`; the predicate `well_formed(g)` ties together monotone `row_ptr`, in-range neighbours, and the relevant `\valid_read` ranges. The signature is `int dfs_check_reachability(const Graph *g, size_t start, size_t target)` and we require `0 <= start < g->n`, `0 <= target < g->n`. Requirements: Return `1` iff there exists a path from `start` to `target`, otherwise `0`; formally `(\result != 0) <==> reachable(g, start, target)`. The natural algorithm pushes `start` onto a stack, repeatedly pops a vertex `u`, marks it visited, and pushes each unvisited neighbour, returning `1` as soon as `target` is popped or pushed and `0` after the stack empties. Verification Challenges: (1) Soundness: when you return `1`, supply a witness path `p[0..plen)` with `is_path(g, p, plen)`, `p[0] == start`, `p[plen-1] == target`. A standard technique is to maintain a ghost `parent[]` array and reconstruct the path. (2) Completeness: when you return `0`, no path exists. The key invariant is "every path from `start` to a not-yet-visited vertex must pass through a vertex currently on the stack"; once the stack empties, all reachable vertices are visited and `target` is not among them. (3) Termination: cycles must not loop the search. Use a loop variant such as `(n - visited_count) * (n + 1) + stack_size` or simpler `n - visited_count` once you prove visited monotonically grows by one per outer iteration. ACSL-specific notes: (a) `reachable(g, start, end)` is defined via `\exists int *p, integer plen; is_path(g, p, plen) && p[0] == start && p[plen - 1] == end`; under the default `Typed` model WP treats this as quantification over phantom buffers, so you must construct or refute the witness with a concrete (ghost) array. (b) The contract uses `assigns \nothing` because internal DFS state lives in local automatic storage; model the visited array and stack with ghost variables so they need not be mentioned in `assigns`. (c) `size_t` parameters compare against `g->n` which is also `size_t`; widen to `integer` inside specs to avoid signed/unsigned mixing. (d) Provide loop invariants tying the stack contents and visited markers to the reachability semantics, e.g. "every vertex on the stack is reachable from `start`" and "every vertex marked visited is reachable from `start`"; these flow directly from the inductive `is_path` definition. (e) Avoid top-level lemmas; the preamble predicates `well_formed`, `has_edge`, `is_path`, `reachable` are sufficient when combined with ghost-code witness constructions inside the function body.
