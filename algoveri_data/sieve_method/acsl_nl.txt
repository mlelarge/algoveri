Your task is to implement the Sieve of Eratosthenes to compute primality for every integer up to `n` and verify its functional correctness in ACSL using Frama-C/WP. Signature: `void sieve_of_eratosthenes(size_t n, bool *primes)`. Preconditions: `0 <= n <= 100000` and `primes` is a writable buffer of length at least `n` (when `n > 0`). The contract specifies `assigns primes[0 .. n - 1]` and the postcondition `\forall integer i; 0 <= i < n ==> ((primes[i] != 0) <==> is_prime(i))` where `is_prime(i) := i > 1 && \forall integer d; 1 < d < i ==> !divides(d, i)` and `divides(d, n) := d != 0 && n % d == 0`. The standard algorithm: initialize `primes[0] = primes[1] = false`, all others `true`; for `i` from 2 while `i*i < n`, if `primes[i]` is true mark every multiple `j` from `i*i` step `i` while `j < n` as composite; the remaining `true` entries are exactly the primes below `n`. Verification challenges: (1) Outer-loop invariant — at iteration `i`, every index `k < n` that has a prime factor strictly less than `i` is already marked false; equivalently, for `k` still true, `k <= 1` or `k`'s smallest prime factor is `>= i`. (2) Inner-loop invariant (`j += i`) — every `j` visited is a multiple of `i` (specifically `j = i*i + t*i` for some `t >= 0`), so marking it false is sound; the loop covers all multiples of `i` in `[i*i, n)`. (3) `i*i` optimization — at the time we process prime `i`, every composite `c < i*i` has a prime factor strictly less than `i` (because any composite has a prime factor `<= sqrt(c) < i`), so it is already marked. (4) Final correctness — after the outer loop, an index is still true iff it has no prime factor below `\sqrt{n}` and is `> 1`; combined with the standard fact that any composite `<= n` has a prime factor `<= \sqrt{n}`, this gives `primes[i] <==> is_prime(i)`. ACSL-specific notes: (a) The original specification had an operator-precedence bug — `\forall i; 0 <= i < n ==> (primes[i] != 0) <==> is_prime(i)` parsed as `(... ==> (primes[i] != 0)) <==> is_prime(i)` because `<==>` binds looser than `==>`. The fixed contract uses explicit parentheses `((primes[i] != 0) <==> is_prime(i))`. (b) Use default `-wp-model Typed`; only one writable buffer, no `\separated` required. (c) Both loops need `loop variant` clauses for termination. (d) `i*i` does not overflow because `n <= 100000` bounds `i <= 316`. (e) No top-level lemmas should be added; ghost lemma functions for "every composite below `i*i` has a small prime factor" are recommended.
