Your task is to implement a deterministic variant of Quick Sort and verify its functional correctness in ACSL using Frama-C/WP. The pivot must always be chosen as the first element of the (sub-)slice being sorted. You need to provide two pieces of code: (1) a partition routine that reorders the slice such that all elements strictly less than the pivot come before it and all elements greater than or equal to the pivot come after it, returning the final index of the pivot, and (2) the recursive quick_sort that uses the partition index to sort sub-ranges. Signature: `void quick_sort(int32_t *a, size_t n)`, with `0 <= n <= 1000` and the array fully writable. The contract specifies `assigns a[0 .. n - 1]` and two postconditions: (a) `is_sorted(a, n)` — non-decreasing — and (b) `same_multiset{Pre,Post}(a, n)` expressed via the recursive logic function `count_occ` (a count-occurrences definition). Verification challenges: (1) Partitioning correctness — proving that after partition the array layout is `[< pivot, pivot, >= pivot]` and that the multiset of values is unchanged. The natural proof uses two-loop invariants stating that the prefix below the scan index contains only elements `< pivot` and the segment between contains only elements `>= pivot`, combined with a frame condition on counts. (2) Termination — with a first-element pivot, a sorted input could yield a recursive call of size n-1 and 0; you must prove the pivot index strictly bounds the recursion by introducing a `loop variant` (or a `decreases` clause on the recursive call) such as `r - l`. (3) Multiset preservation across recursive calls — the strongest invariant; it requires showing that swapping two cells does not change `count_occ`, and that concatenating two sorted halves around the pivot preserves the global multiset. ACSL-specific notes: (a) The provided axiomatic logic function `count_occ` is defined inductively over n; no top-level lemmas are required — but you may need lemma functions (proved by induction with ghost loops) for `count_occ` invariance under swap and over concatenation. (b) Use the default `-wp-model Typed`; no separation clauses are needed because there is a single array. (c) `assigns a[0 .. n - 1]` is mandatory; the recursive helper should restrict its `assigns` to its sub-range so the frame across the two halves composes cleanly. (d) Avoid writing axioms beyond the given preamble — ACSL lemmas must be discharged, not assumed. (e) Termination on size_t arithmetic is fine since indices stay in [0, n] and n is bounded by 1000.
