Your task is to implement the Karatsuba algorithm for fast polynomial multiplication and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: inputs `a` and `b` are `int64_t` arrays of equal length n, with 0 < n, 2*n <= 1000, `is_power_of_2(n)`, and every coefficient bounded by `coeffs_bounded(.., n)` to lie in [-1_000_000, 1_000_000]. The output `out` has length `2*n - 1` and `\valid(out + (0 .. 2*n - 2))`. Requirements: implement `poly_multiply(const int64_t *a, int n, const int64_t *b, int64_t *out)` so that `out[k] == spec_poly_mul_coeff(a, n, b, n, k)` for every k in [0, 2n - 1). (1) Algorithm: split a into halves A0 = a[0..m), A1 = a[m..n) where m = n/2 (similarly B0, B1). Recursively compute z0 = A0*B0, z2 = A1*B1, and z1 = (A0 + A1)*(B0 + B1). Then the result is `out[k] = z0[k] + (z1[k - m] - z0[k - m] - z2[k - m]) + z2[k - 2m]` with the appropriate range checks. The base case (n == 1) is the scalar product `out[0] = a[0] * b[0]`. (2) Functional Correctness: the postcondition uses the `spec_poly_mul_coeff` logic function, defined via `spec_convolution_sum` recursing on the inner index i — base case 0 when i < 0, step case `((0 <= i < na && 0 <= k - i < nb) ? a[i] * b[k - i] : 0) + spec_convolution_sum(.., i - 1)`. Verification Challenges: (1) Algebraic equivalence — proving `(A0 + A1)*(B0 + B1) - A0*B0 - A1*B1 == A0*B1 + A1*B0` and lifting this identity to convolutions over half-length arrays needs a ghost lemma (kept inside the function body, not as a top-level axiom) that splits `spec_convolution_sum(a, n, b, n, k, k)` into four sub-convolutions indexed by m. (2) Overflow safety — A0[i] + A1[i] fits in i64 (sum of two values each <= 10^6), and (A0+A1) has length n/2 <= 500, so z1[j] is bounded by 500 * (2*10^6)^2 = 2*10^15, well below i64::MAX. The proof obligation must instantiate this bound as part of the invariant. (3) Recursion termination — `decreases n` since each recursive call halves n; the base case n == 1 must be unfolded explicitly. ACSL-specific notes: (a) The contract requires `\separated(out + (0 .. 2*n - 2), a + (0 .. n - 1), b + (0 .. n - 1))` so writes to out do not invalidate the predicate reads from a and b — without this, `spec_convolution_sum`'s `reads` clauses cannot be framed. (b) Recursive Karatsuba needs scratch buffers for A0+A1, B0+B1, z0, z1, z2 — declare them as VLAs or pass a workspace pointer; each must be `\separated` from a, b, out. (c) Use `loop variant` on the linear copy/add passes and `decreases n` on the recursion. (d) Do not introduce free-standing axioms; the algebraic identity must be discharged by ghost code that unfolds spec_convolution_sum_step. (e) `-wp-model Typed` with the `\separated` clause in place is sufficient; the i64 multiplications discharge against the explicit numeric bound stored in the invariant.
