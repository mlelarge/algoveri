Your task is to implement the Coin Change algorithm (finding minimum number of coins) and verify its correctness in Dafny. You are given an integer array `coins` representing coins of different denominations and an integer `amount` representing a total amount of money. Return the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return -1. You may assume that you have an infinite number of each kind of coin. In the incomplete code, the specification defines what constitutes a valid change combination (counts of each coin that sum to amount) and the metric to minimize (total count of coins). Your task is to implement the algorithm (typically DP) and prove: (1) Lower Bound: Any valid combination has a total coin count greater than or equal to the result (if result != -1). (2) Existence: If result != -1, there exists a valid combination with exactly that many coins. (3) Impossibility: If result == -1, no valid combination exists.
