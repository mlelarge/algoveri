Your task is to implement the Knuth-Morris-Pratt (KMP) string-matching algorithm to find every occurrence of a needle within a haystack and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: `haystack` and `needle` are read-only `unsigned char` buffers of lengths `hay_len` and `needle_len`, both in `[0, 1000000)` (so index arithmetic stays within `int`). The output buffer `out_indices` is sized for at most `hay_len + 1` indices and `out_len` is a single `int`. Requirements: Implement `void kmp_search(const unsigned char *haystack, int hay_len, const unsigned char *needle, int needle_len, int *out_indices, int *out_len)`. Write all matching start indices into `out_indices[0 .. *out_len - 1]` and store the count in `*out_len`. Both Soundness (every emitted index is a real match: `matches_at(...)`) and Completeness (every match index is emitted somewhere in `out_indices[0 .. *out_len)`) must be proven. Algorithm: (1) Preprocess the longest-proper-prefix-suffix (LPS) array `lps[0..needle_len)` so that `lps[i]` is the length of the longest proper prefix of `needle[0..i]` that is also a suffix. (2) Sweep `haystack` with two indices `i, j` advancing `i` per haystack character, advancing `j` on character match, and using `j = lps[j - 1]` on mismatch when `j > 0`. Verification Challenges: (1) LPS Property: prove that the constructed LPS table satisfies its mathematical definition; carry an inline ghost lemma asserting that for every `i`, `needle[0..lps[i]] == needle[i + 1 - lps[i]..i + 1]`. (2) Safe Skipping: when LPS sends `j` backwards, prove that no match is missed in the skipped range; this requires an invariant of the form "for every `k` in (last_emit, i - j], `!matches_at(..., k)`". (3) Loop Invariants: relate `j` to the longest prefix of `needle` matching the haystack ending at `i - 1`. ACSL-specific notes: (a) The contract requires `\separated(out_len, out_indices + (0 .. hay_len))` so that the final `*out_len = count;` write does not alias an `out_indices[k]` cell — without this the soundness/completeness postconditions cannot be discharged. (Verus's borrow-checker provides this guarantee implicitly via `&mut` references; in ACSL it must be stated.) (b) The `unsigned char` element type means each character lives in `[0, 255]`; the predicate's pointer-to-character casts use `(unsigned char *)` so `\valid_read` ranges align with the predicate's reads. (c) `assigns out_indices[0 .. hay_len], *out_len;` covers both writes. (d) Loop variants: outer sweep variant `hay_len - i`; LPS-fallback inner loop variant `j`. (e) The default `Typed` memory model is sufficient with the `\separated` clause in place. (f) Do not introduce top-level lemmas; carry LPS-property reasoning inline through ghost asserts at the points where `j` is updated.
