Your task is to implement the `pop_max` operation for an array-backed binary max-heap and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the heap storage is the prefix `a[0 .. *len - 1]` of a uint32_t array, with 0 < *len <= 1023, satisfying the heap predicate `is_heap(a, *len)`: every child a[i] (1 <= i < *len) is at most its parent a[(i - 1) / 2]. Requirements: implement `pop_max(uint32_t *a, int *len)` returning the old root a[0] and shrinking the heap by one. (1) Algorithm: save the root value `max = a[0]`, move the last element a[*len - 1] into a[0], decrement *len, then "sift down" the new root by repeatedly swapping it with the larger of its two children until the heap property is restored. (2) Functional Correctness: prove `is_heap(a, *len)` holds after the swap-down (Structural Integrity), `*len == \old(*len) - 1`, that the returned `max` is at least every element of the original heap (`\forall x. count_occ{Pre}(a, \old(*len), x) > 0 ==> \result >= x`), and that the post-state multiset equals the pre-state multiset minus exactly one occurrence of \result (the count_occ equation in the postcondition). Verification Challenges: (1) Multiset preservation — sift-down performs a sequence of swaps; each swap only exchanges two cells, but the global count_occ predicate must be carried through the loop. The standard pattern (used in `bubble_sort`'s ACSL proof) is a ghost loop that re-establishes `count_occ{Here}(a, *len, x) == count_occ{Pre}(a, \old(*len), x) - (x == \result ? 1 : 0)` after each swap by appealing to the count_occ_step axiom symmetrically on both sides. (2) Structural integrity — the inner sift-down loop invariant must say "the heap property holds everywhere except possibly at index `i`, and the relative parent at (i-1)/2 already dominates a[i]", which needs careful indexing on (2i+1, 2i+2). (3) Maximality of \result — follows from the heap invariant on the *original* array (root dominates everything), but the proof must reference the {Pre} label explicitly. ACSL-specific notes: (a) The contract requires `\separated(a + (0 .. *len - 1), len)` so that decrementing *len and the cell writes do not alias (Verus's `&mut self` field decomposition gives this implicitly). (b) Use `loop variant *len - i` for the sift-down. (c) The `count_occ` axiomatic preamble is the only logic state needed — do not add free-standing lemmas; instead prove the multiset step inside ghost code attached to each swap. (d) `-wp-model Typed` with the `\separated` clause is sufficient; expect Alt-Ergo to discharge arithmetic, while count_occ instantiations may benefit from ghost `assert`s.
