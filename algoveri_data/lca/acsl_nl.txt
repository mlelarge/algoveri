Your task is to implement the Lowest Common Ancestor (LCA) function for a binary tree with distinct node values and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: You are given a `const TreeNode *root`; each `TreeNode` carries `uint64_t val` and pointers `left`, `right`. The contract requires `tree_contains(root, p)` and `tree_contains(root, q)` (both target values exist somewhere in the tree) and `tree_distinct(root)` (all values are pairwise distinct, recursively defined as: left and right subtrees are distinct, the node's own value is not in either subtree, and left and right value sets are disjoint). Requirements: Implement `lca` returning a `uint64_t` equal to the value of the deepest common ancestor of `p` and `q`. (1) Algorithm: The classical recursive procedure: if `root == NULL`, return a sentinel; if `root->val == p` or `root->val == q`, return `root->val`; otherwise recurse into both children. If both calls return non-trivial answers, the current node is the LCA; otherwise return whichever side returned a hit. Because both `p` and `q` are guaranteed to be in the tree (by precondition), the function always returns a meaningful value (the C signature returns a plain `uint64_t`, mirroring the unwrapped Verus `Option<u64>`). (2) Functional Correctness: The two postconditions are `is_common_ancestor(root, \result, p, q)` (the returned value's subtree contains both `p` and `q`) and `\forall integer x; is_common_ancestor(root, x, p, q) ==> spec_get_depth(root, x) <= spec_get_depth(root, \result)` (`\result` is at maximal depth among common ancestors). The axiomatic block `LcaSpec` defines `spec_get_subtree(t, target)` (the subtree rooted at the node with value `target`, or `\null`) and `spec_get_depth(t, target)` (depth, with `-1` denoting absence) recursively. Verification Challenges: (1) Depth-Maximality by Structural Induction: The maximality postcondition - that no other common ancestor lies deeper than `\result` - inherently requires induction over the tree shape, since arbitrary `x` could appear anywhere in `root`'s subtree. ACSL has no built-in structural induction, so this proof must be staged as a series of ghost lemmas, each proved by hand using the axiomatic definitions. This is the hardest verification obligation in the file and is at the frontier of what WP can discharge without significant manual scaffolding. (2) Recursive Function Verification: The implementation must be verified recursively, with a contract on each recursive call asserting that the returned value satisfies a partial common-ancestor property at the current subtree, then composed at the parent. (3) Reads-Clause Hygiene for Logic Functions: `spec_get_subtree` and `spec_get_depth` declare `reads t, t->val, t->left, t->right;` - a single-level read; you may need an inductive lemma that extends this to the entire reachable subtree to use the spec's recursive structure. ACSL-specific notes: (a) The contract is `assigns \nothing;` because the function is pure; no `\separated` is needed for read-only traversal. (b) Tree pointers are not aliased in valid trees, but ACSL does not know this - you may need a precondition or ghost invariant `tree_acyclic(root)` (every node is reachable along a unique path) to discharge the recursive specification. (c) The C signature returns `uint64_t` directly rather than an `Option`-like wrapper; this is sound because the precondition `tree_contains(root, p) && tree_contains(root, q)` guarantees an answer exists. (d) `-wp-model Typed` is sufficient. (e) No top-level lemmas are required by the contract, but `lemma spec_get_depth_is_some_when_contained` (depth is `>= 0` exactly when `node_contains` holds) is the core auxiliary needed to bridge `is_common_ancestor` and `spec_get_depth`. (f) The use of `\let` inside axioms is supported in modern Frama-C; the `spec_get_subtree_node` axiom uses nested `\let` to express "look in left subtree, else right" without duplicating sub-expressions.
