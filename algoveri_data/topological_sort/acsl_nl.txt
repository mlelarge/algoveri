Your task is to implement Topological Sort on a directed graph and verify its functional correctness in ACSL using Frama-C/WP. The graph is encoded in CSR (Compressed Sparse Row) form: `typedef struct { size_t n; const size_t *row_ptr; const size_t *col_idx; } Graph` with `row_ptr` of length n+1 and `col_idx` of length `row_ptr[n]`; edges out of vertex v are `col_idx[row_ptr[v] .. row_ptr[v+1] - 1]`. Requirements: Implement `int topological_sort(const Graph *g, size_t *order)` returning 1 on success (and writing a topological order into order[0..n-1]) or 0 if the graph has a cycle. (1) Algorithm: use either Kahn's algorithm (repeatedly remove a zero-in-degree vertex) or DFS-based topological sort with gray/black cycle detection. Kahn's is more naturally framed for WP because the loop bound is the count of processed vertices. (2) Functional Correctness: when result is 1, prove `!graph_has_cycle(g)` and `is_topological_ordering(g, order)`, the latter saying order is a permutation of [0, n) with every edge u->v having u at a strictly smaller index than v. When result is 0, prove `graph_has_cycle(g)` — the graph really does contain a cycle, expressed via the existential `\exists int *p, integer plen; plen > 0 && \valid_read(p+(0..plen-1)) && is_cycle(g, p, plen)`. Verification Challenges: (1) Cycle witness construction: the result == 0 case is the dominant difficulty. Returning 0 requires actually exhibiting a cycle as a vertex sequence with `p[0] == p[plen-1]` and edges throughout. In Kahn's algorithm, when the in-degree-zero queue empties before processing all vertices, every remaining vertex has positive in-degree; walking predecessors from any such vertex must eventually revisit one, giving the witness — but it must be materialized as a ghost array satisfying `\valid_read`. In DFS, the gray-stack at the moment a back-edge is encountered IS the cycle witness; carry it as a ghost. (2) Permutation invariant on success: the loop invariant must say "the prefix order[0..k] contains k distinct vertices and every edge into a prefix vertex has its source also in the prefix"; quantifier instantiation for the no-backwards-edge postcondition is non-trivial. (3) Termination: variant is `n - k` where k counts processed vertices. ACSL-specific notes: (a) The contract uses `assigns order[0 .. g->n - 1];`; ghost arrays for in-degree, color, and the cycle witness must live in `\ghost` blocks. (b) Use `-wp-model Typed`; the CSR arrays are read-only (`const`), so no separation between row_ptr and col_idx is required. (c) Allocating the cycle witness array (size at most n+1 for a simple cycle) within WP's allocation model is the chief practical obstacle and should be flagged. (d) No top-level lemmas should be required; the predicates in the preamble (well_formed, has_edge, is_path, is_cycle, graph_has_cycle, is_topological_ordering) are sufficient.
