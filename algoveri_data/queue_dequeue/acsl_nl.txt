Your task is to implement the dequeue (pop-front) operation of a verifiable Queue and verify its FIFO correctness in ACSL using Frama-C/WP. Preconditions: the queue is encoded by `int32_t *data` and `size_t *len`, with the live view being `data[0 .. *len - 1]`. The contract requires `\valid(len)`, `0 < *len <= 1024`, and `\valid(data + (0 .. *len - 1))`. Requirements: implement `int32_t queue_dequeue(int32_t *data, size_t *len)` returning the front element and shifting the rest left by one. (1) Algorithm: save `result = data[0]`, then for i from 1 to *len - 1 set data[i - 1] = data[i], and finally decrement *len. (2) Functional Correctness: the contract states three obligations — `\result == \old(data[0])` (correct returned value), `*len == \old(*len) - 1` (size decreases by one), and `\forall integer i; 0 <= i < *len ==> data[i] == \old(data[i + 1])` (every remaining element equals the next-older element from the input). Verification Challenges: (1) Shift loop invariant — at iteration i the prefix data[0 .. i - 1] already equals \old(data[1 .. i]), and the suffix data[i .. \old(*len) - 1] still equals \old(data[i .. \old(*len) - 1]). Frama-C/WP needs both halves stated explicitly; otherwise the postcondition's universal quantifier cannot be discharged. (2) Capturing \old values across pointer writes — every read on the right-hand side of the invariant must be guarded with `\at(.., Pre)` so that the iteration doesn't loop-carry stale data. (3) Memory framing — the `assigns data[0 .. *len - 2], *len;` clause means the cell `data[*len - 1]` is unspecified after the call, and the invariant must allow it. ACSL-specific notes: (a) The contract requires `\separated(data + (0 .. *len - 1), len)` so the final `*len = *len - 1` write does not alias any data cell — without this, the universal quantifier over data[i] post-state cannot be discharged (Verus's `&mut data` and `&mut len` give this for free, but in ACSL it must be stated). (b) Use `loop variant *len - i` for termination. (c) The contract is otherwise self-contained; no axiomatic preamble is required because the queue's view is just `data[0 .. *len - 1]` and FIFO order is captured by the index-shift quantifier. (d) `-wp-model Typed` is sufficient; expect Alt-Ergo to discharge the invariant once `\separated` and the prefix/suffix decomposition are in place.
