Your task is to implement the Aho-Corasick multi-pattern string-search algorithm and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The haystack `haystack` (an `unsigned char *` of length `hay_len < 1,000,000`) and the pattern set (parallel arrays `patterns: const unsigned char *const *` and `pat_lens: const int *` of length `pat_count`, with `0 < pat_count < 1,000,000` and each individual pattern shorter than 1,000,000) are read-only. Output buffers `out_pids` and `out_idxs` of length `max_results` collect the matches and must be pairwise `\separated` and separated from the haystack and `pat_lens`. Requirements: Implement `ac_automata_search(...)` returning `int` — the number of matches written into the parallel output arrays (`<= max_results`). (1) Trie Construction: Build a trie from the pattern set; each pattern terminator stores its pattern id. (2) Failure Links: BFS the trie to compute, for each node, the longest proper suffix that is also a prefix of some pattern (the failure link). (3) Traversal: Scan the haystack once, advancing through trie children where possible and following failure links on mismatch; at every node, walk failure-link ancestors to emit all matches that end at the current text position. (2) Functional Correctness: Soundness: every emitted `(out_pids[k], out_idxs[k])` satisfies `0 <= out_pids[k] < pat_count` and `matches_at(haystack, hay_len, patterns[out_pids[k]], pat_lens[out_pids[k]], out_idxs[k])`. Completeness: every `(pid, idx)` with `matches_at(...)` true appears somewhere in the output (up to `\result`). The preamble defines `matches_at` (textbook substring match) and `well_formed_patterns` (each pattern is bounded and readable). Verification Challenges: (1) Automaton invariant: failure links correctly point to the longest suffix-prefix match. (2) State validity: after processing `haystack[0..i]`, the current trie node represents the longest suffix of `haystack[0..i]` that is a prefix of some pattern. (3) Multi-Pattern completeness: every match (including substring matches reachable only via failure-link chains) is emitted. ACSL-specific notes: (a) Several `\separated` clauses were added: `out_pids` vs `out_idxs`, each output buffer vs `haystack`, and each output buffer vs `pat_lens`. Verus's borrow checker provides this disjointness implicitly via `&Vec<u8>` for the input and a fresh `Vec<(usize, usize)>` for the output; in ACSL it must be stated explicitly. We do not assume separation against the per-pattern `patterns[i]` buffers because that would require a quantified `\separated`; the contract reads `pat_lens` and `patterns[i]` only for soundness/completeness postconditions, not concurrently with writes. (b) The contract uses parallel arrays `(out_pids[k], out_idxs[k])` instead of Verus's `Vec<(usize, usize)>` because ACSL has no native tuples. (c) The trie and failure-link tables are local arrays — no heap allocation is required, just stack arrays sized by total pattern length plus one. Provide `loop variant` for the BFS construction (variant `nodes_remaining`), the trie construction (variant `pat_lens[pid] - i`), and the haystack scan (variant `hay_len - i`). (d) The default `-wp-model Typed` memory model is sufficient given the new `\separated` clauses. (e) Quantifier instantiation for completeness is the deepest part of the proof — supply ghost code recording, for each text position, which trie state was reached, then conclude that every match of every pattern was visited. (f) Per project rules, no top-level lemmas or axioms in the user solution; supplement reasoning with `assert` and `loop invariant` only.
