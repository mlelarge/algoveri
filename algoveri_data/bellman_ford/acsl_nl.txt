Your task is to implement the Bellman-Ford single-source shortest-path algorithm and verify its functional correctness in ACSL using Frama-C/WP. Unlike Dijkstra, Bellman-Ford handles negative edge weights and detects negative cycles. Preconditions: The graph is a CSR-encoded `WeightedGraph` (struct with `n`, `row_ptr`, `col_idx`, and `col_weight`) satisfying `well_formed(g)` (a simple graph: no parallel edges from `u` to the same `v`) and `weights_and_size_bounded(g)` (`g->n <= 100,000`, edge weights in `[-100,000, 100,000]`, ensuring path totals fit in `int64_t`). The start index is in `[0, g->n)`. Output buffers `dist` (`int64_t`) and `reachable` (`int`) each have length `g->n` and must be pairwise `\separated` and separated from the graph CSR arrays. Requirements: Implement `bellman_ford(const WeightedGraph *g, size_t start, int64_t *dist, int *reachable)` returning `int`: `1` on success and `0` if a reachable negative cycle exists. (1) Algorithm: Initialize `dist[start] = 0`, mark all vertices unreachable. For `g->n - 1` iterations, relax every edge `(u, v, w)`: if `reachable[u]` and (`!reachable[v]` or `dist[u] + w < dist[v]`), set `dist[v] = dist[u] + w` and `reachable[v] = 1`. Then run a final relaxation pass: if any edge can still be relaxed, return `0`. Otherwise return `1`. (2) Functional Correctness: On `\result == 1`, every `v` with `reachable[v] == 1` satisfies `is_shortest_dist(g, start, v, dist[v])` and every `v` with `reachable[v] == 0` satisfies `unreachable_from(g, start, v)`. On `\result == 0`, `has_negative_cycle(g)` holds. The preamble defines `has_edge`, `connected`, `is_path`, `path_weight` (in axiomatic block `BellmanFordPathWeight` with base `plen < 2 ==> path_weight == 0` and step `path_weight(plen) == path_weight(plen-1) + edge_weight(p[plen-2], p[plen-1])`, with `edge_weight` being uniquely defined thanks to the simple-graph constraint), `is_shortest_dist`, `unreachable_from`, and `has_negative_cycle`. Verification Challenges: (1) Relaxation invariant: after `k` outer iterations, for every `v`, `dist[v]` is the shortest path weight using at most `k` edges (or `reachable[v] == 0` means no such path of length `<= k` exists). (2) Negative-cycle detection: if relaxation succeeds in iteration `g->n`, exhibit a closed walk with negative total weight. The standard argument uses the pigeonhole principle on vertices visited along the back-pointer chain. (3) Upper-bound correctness: `dist[v]` always corresponds to *some* real path from `start` to `v` (not just an arbitrary number). ACSL-specific notes: (a) The fix added `\separated` clauses between `dist`, `reachable`, and the graph CSR arrays (`g`, `g->row_ptr`, `g->col_idx`, `g->col_weight`) so writes to the output buffers cannot perturb the graph — Verus's borrow rules guarantee this implicitly. (b) The axiomatic block `BellmanFordPathWeight` provides `edge_weight` and `path_weight` as logic functions with `reads` clauses; their `axiom`s are your only handle on path arithmetic. The recursive step axiom is what lets you "unroll" `path_weight` by one edge inside `assert` chains. (c) Use `loop variant g->n - 1 - k;` for the outer relaxation loop and `g->row_ptr[g->n] - e_count` for the inner edge scan; both are decreasing. (d) The default `-wp-model Typed` memory model is sufficient. (e) The `i64`/`int64_t` arithmetic is bounded by `g->n^2 * max_weight = 100,000 * 100,000 = 10^10`, well within `int64_t` range — but explicitly state `\at(dist[u], LoopEntry) + w` does not overflow inside the relaxation invariant. (f) Per project rules, no top-level lemmas or axioms beyond the existing preamble; supplement reasoning with `assert` and `loop invariant` annotations only.
