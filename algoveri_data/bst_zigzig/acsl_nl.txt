Your task is to implement the Zig-Zig (double right rotation) operation on a Binary Search Tree and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input pointer `g` is valid for reading and satisfies `is_bst(g)`, with `g->left != \null`, `g->left` valid for reading, and `g->left->left != \null`. The Zig-Zig case applies when grandchild `X = g->left->left`, parent `P = g->left`, and grandparent `G = g` form a straight (left-left) line. Requirements: Implement `bst_zig_zig(BstNode *g)` returning a non-null `BstNode *` whose root value equals the original `g->left->left->val` (i.e., `X` becomes the new root). Pictorially, given `g = (G, P = (P, X = (X, A, B), C), D)`, the result is `(X, A, (P, B, (G, C, D)))`. (1) Algorithm: This is two right-rotations in sequence: first rotate the edge between `G` and `P` (Zig at `g`), then rotate the edge between `P` and `X` (Zig at the new right child). Equivalently, allocate (or re-link) so that `X` is at the top, with subtree `A` on the left, and `(P, B, (G, C, D))` on the right. (2) Functional Correctness: Postconditions: `\result != \null`, `is_bst(\result)`, view-equality `\forall integer x; in_view(\result, x) <==> in_view{Pre}(g, x)`, and `\result->val == \at(g->left->left->val, Pre)` (`X` is the new root). The preamble provides `in_view` and `is_bst` (recursive predicates). Verification Challenges: (1) BST invariant preservation: in the rotated tree the root is `X`, with subtree `A` on the left (every value `< X` in original BST); on the right is `(P, B, (G, C, D))`. The proof requires showing: every value in `B` is in `(X, P)` (i.e., `> X` and `< P`), every value in `C` is in `(P, G)`, and every value in `D` is `> G`. All these follow from the original BST invariants, but writing them out as explicit ACSL `assert` chains is laborious. (2) Set Preservation: the value multiset is preserved by re-parenthesization. (3) Subtree juggling across three levels: this is the hardest BST rotation to verify because four subtrees (`A`, `B`, `C`, `D`) are simultaneously re-parented. ACSL-specific notes: (a) The contract is `assigns \nothing;` — the implementation allocates fresh nodes for `X`, `P`, and `G` in the new shape, leaving the original tree intact. If you prefer in-place pointer manipulation, you must relax `assigns` to a list of mutated fields. (b) `\at(g->left->left->val, Pre)` captures `X`'s value at the entry point, before any rebuild. (c) `is_bst` is recursive; supply `decreases` on call sites or invoke on subtrees of known ghost-height. (d) The default `-wp-model Typed` memory model is sufficient. (e) The cleanest verification approach is to explicitly bind ghost variables `X_v`, `P_v`, `G_v` and ghost subtree pointers `A`, `B`, `C`, `D` at function entry, then state intermediate `assert` clauses describing what each of the new nodes contains, and finally chain to `in_view{Pre}(g, x) <==> in_view(\result, x)`. (f) Per project rules, no top-level lemmas or axioms in the user solution; everything must live inside the function.
