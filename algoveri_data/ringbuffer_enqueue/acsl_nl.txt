Your task is to implement and verify the enqueue (push-back with overwrite) operation for a fixed-capacity Ring Buffer in ACSL using Frama-C/WP. In C the buffer is represented as four arguments: `int32_t *data`, `size_t *head`, `size_t *len`, and `size_t cap`. The live view is `data[(*head + i) % cap]` for `0 <= i < *len`. Signature: `void ringbuffer_enqueue(int32_t *data, size_t *head, size_t *len, size_t cap, int32_t value)`. Preconditions: `0 < cap <= 1024`, `data` valid over `[0, cap-1]`, `head` and `len` valid pointers, `0 <= *head < cap`, `0 <= *len <= cap`. The contract specifies `assigns data[0 .. cap - 1], *head, *len;` and four postconditions covering the two cases: (a) Not full — `*len_pre < cap`: length grows by one, head is unchanged, the new value is appended at logical index `*len_pre`. (b) Full — `*len_pre == cap`: length stays at cap, head advances by one mod cap (effectively dropping the oldest element), the new value is written at the slot just freed. (The given contract states the head/length transitions; you should also prove that the slot `data[(\at(*head,Pre) + \at(*len,Pre)) % cap]` holds `value` afterwards, and that all other live cells are preserved with the appropriate index shift.) Verification challenges: (1) Two-case reasoning over fullness — your implementation likely branches on `*len == cap`; the contract postconditions are conjuncts guarded by `\at(*len, Pre) ... cap` and each conjunct must be discharged independently. (2) Modular arithmetic for the write index — `(head + len) % cap` for the not-full case, and `(head + cap) % cap` for the overwrite case. (3) Frame condition on untouched cells — although `assigns data[0 .. cap - 1]` is permissive, the postcondition implicitly requires that exactly one cell is overwritten (the new tail slot); WP discharges this from the actual write. (4) Aliasing of `head`, `len` and `data`. ACSL-specific notes: (a) The contract requires `\separated(head, len, data + (0 .. cap - 1))` so that the writes to `*head`, `*len`, and `data[k]` do not interfere — Verus's `&mut` references guarantee disjointness, in ACSL this must be stated. (b) Use the default `-wp-model Typed`. (c) No axiomatic preamble or top-level lemmas are required. (d) No loop, hence no loop variant. (e) `size_t` is treated as non-negative, so the `% cap` operation is mathematically clean. (f) When implementing, be careful that the not-full and full branches both establish all four postconditions, including the trivial ones for the opposite branch (vacuously true under their guards).
