Your task is to implement the Binary Search algorithm (specifically the lower_bound variant) and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input array `a` of `int32_t` is sorted in non-decreasing order and has length `n` (a `size_t`) with `n <= 0x7FFFFFFF`. If `n > 0` the buffer `a + (0..n-1)` must be readable. The target is a single `int32_t`. Requirements: Implement the function `binary_search_lower_bound` which returns a `size_t` and finds the first index `k` such that `a[k] >= target` in O(log n) time using a shrinking window. (1) Algorithm: Maintain two indices `low = 0` and `high = n`; while `low < high`, compute `mid = low + (high - low) / 2` (avoiding overflow), then either set `low = mid + 1` (if `a[mid] < target`) or `high = mid` (otherwise). Return `low`. (2) Functional Correctness: You must verify the three postconditions: `0 <= \result <= n`, `\forall integer i; 0 <= i < \result ==> a[i] < target`, and `\forall integer i; \result <= i < n ==> a[i] >= target`. The preamble defines the `is_sorted` predicate via `\forall integer i, j; 0 <= i <= j < n ==> a[i] <= a[j]`. Verification Challenges: (1) Two-sided invariant: Inside the loop you must maintain (a) all elements in `a[0..low)` are strictly less than the target, and (b) all elements in `a[high..n)` are greater than or equal to the target. Together with `low <= high <= n` these invariants establish the postcondition once `low == high`. (2) Termination: The shrinking window `high - low` strictly decreases each iteration; supply a `loop variant high - low;`. ACSL-specific notes: (a) The contract uses `assigns \nothing;` because the search is a pure read-only computation. (b) The `is_sorted` predicate is the only predicate in the preamble — no axiomatic blocks or recursive logic functions are required. (c) The midpoint computation must be written in C as `low + (high - low) / 2` to keep WP happy that no `size_t` overflow happens (otherwise WP will flag the implicit overflow check on `(low + high) / 2`). (d) The default `-wp-model Typed` memory model is sufficient since there is no aliasing — `a` is read but never written, and `\result` is a scalar. (e) The bound `n <= 0x7FFFFFFF` keeps `mid` within `int32_t` range so signed/unsigned mixing in helper indices is safe. (f) Per project rules, do not introduce any top-level `lemma` or `axiom` outside the existing preamble; all reasoning must live in `loop invariant` / `loop variant` / `assert` annotations inside the function body.
