Your task is to implement Tarjan's Strongly Connected Components algorithm and verify its functional correctness in ACSL using Frama-C/WP. The graph is represented in CSR form via a `Graph` struct (`size_t n; const size_t *row_ptr; const size_t *col_idx;`) — `row_ptr[v]` is the start of vertex `v`'s adjacency list inside `col_idx`, and `row_ptr[n]` is the total edge count. Signature: `int find_sccs(const Graph *g, int *comp)`; the function writes a component id `comp[v]` for each vertex and returns the number of components. Preconditions (given via the `well_formed` and `size_bounded` predicates): graph is well-formed (CSR-consistent, non-decreasing `row_ptr`, every `col_idx[e] < n`), `n <= 1000`, and `comp` is fully writable. The contract specifies `assigns comp[0 .. g->n - 1]`, `0 <= \result <= g->n`, and the validity predicate `is_valid_scc_result(g, comp, \result)` which packages three properties: (i) every `comp[v]` lies in `[0, \result)`, (ii) Strong Connectivity — vertices sharing a component id are mutually reachable via valid paths in the graph, (iii) Maximality — vertices with different component ids are not mutually reachable. Verification challenges: (1) DFS bookkeeping — Tarjan's algorithm maintains `index`, `lowlink`, and `on_stack` arrays plus an explicit DFS stack. You must encode the DFS recursion with explicit bounds and prove that `lowlink[u]` equals the minimum `index` reachable from `u`'s subtree by tree edges plus at most one back/cross edge to a node still on the stack. (2) Stack property — when `lowlink[u] == index[u]`, the nodes popped from the auxiliary stack down to and including `u` form exactly one SCC. This is the hardest invariant: it requires a coupling between the reachability predicate `has_path` (defined existentially over arbitrary path arrays) and the DFS state. (3) Maximality — proving that no two distinct components are mutually reachable needs a global inductive argument over DFS finish times, which Frama-C/WP cannot discharge automatically; you will need extensive ghost state and lemma functions. (4) Path predicate — `has_path` is encoded via an existential over an arbitrary `int *p, integer plen` representing a path; proving such an existential requires constructing the path explicitly. ACSL-specific notes: (a) The graph is read-only; `assigns comp[0 .. g->n - 1]` is the only write set. (b) Use default `-wp-model Typed`; no `\separated` constraints needed because `comp` is the only writable buffer. (c) `has_path` and `path_valid` quantify over arbitrary external `int *p` pointers — discharging existentials requires explicit ghost-array witnesses. (d) This problem is SMT-hard, not a translation defect: strong-connectivity + maximality postconditions are research-level proof obligations, and the path-existential format makes Alt-Ergo and Z3 struggle with quantifier instantiation. (e) No top-level lemmas may be added; helper lemma functions discharged by induction are the recommended route.
