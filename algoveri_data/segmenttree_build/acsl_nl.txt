Your task is to implement the `build` operation for a Range-Maximum Segment Tree and verify its functional correctness in ACSL using Frama-C/WP. The tree is represented as a heap-allocated structure `SegNode { uint64_t val, low, high; SegNode *left, *right; }`. Signature: `SegNode *seg_build(uint64_t l, uint64_t r)`; the function recursively constructs a balanced tree covering `[l, r)` initialized to all zeros and returns its root. Preconditions: `l < r` (non-empty range). The contract specifies `assigns \nothing`, `allocates \result`, `\result != \null`, the structural invariant `is_segment_tree(\result)`, the boundary equalities `\result->low == l && \result->high == r`, and the value postcondition: every index `k` in `[l, r)` lies in the view and `view_at(\result, k) == 0`. The recursive construction: leaf when `l + 1 == r` (a single-cell node with `val = 0`); otherwise compute `mid = (l + r) / 2`, recursively build `[l, mid)` and `[mid, r)`, and return a fresh internal node whose `val` is the max of its children. The preamble defines `is_segment_tree` (a recursive predicate covering both leaf and internal cases with structural bounds and the max property), `in_view(t, k)` for membership, and a logic function `view_at(t, k)` defined by three axioms (leaf, descend-left, descend-right). Verification challenges: (1) Termination of the recursive build — the natural measure is `r - l`, strictly halved on each recursive call; in ACSL the recursive call needs an `assigns \nothing; allocates ...` contract that composes through the recursion. (2) Structural correctness — proving `is_segment_tree(\result)` requires showing that the children's `low/high` line up exactly with `(l + r) / 2`, and that `\result->val` equals the max of children's vals (both are 0 here, so trivially 0). (3) View postcondition — proving `view_at(\result, k) == 0` for every `k` in `[l, r)` requires using the `view_at_left`/`view_at_right` axioms to descend recursively and the `view_at_leaf` axiom at the base. ACSL-specific notes: (a) `view_at` is an axiomatic logic function with three case axioms; discharging the universal `\forall integer k` view postcondition requires careful trigger instantiation, possibly via a ghost induction on `r - l`. (b) `allocates \result` declares the returned pointer as fresh memory, paired with `assigns \nothing`. (c) `is_segment_tree` is a structurally-well-founded recursive predicate; WP accepts the fixpoint. (d) Use default `-wp-model Typed`; freshly malloc'd nodes are auto-separated. (e) `(l + r) / 2` does not overflow `uint64_t` for typical inputs; consider adding `requires l + r <= UINT64_MAX` if needed. (f) This problem is SMT-hard mainly due to the universally-quantified view postcondition, not a translation defect; no top-level lemmas may be added.
