Your task is to implement the Find operation with Path Compression for a Union-Find (Disjoint Set Union) data structure and verify its functional correctness in ACSL using Frama-C/WP. The structure is encoded as `typedef struct { size_t *parent; size_t *rank; size_t n; } UnionFind`, with the validity predicate `is_valid_uf` requiring the rank-strict-monotone DAG property `parent[i] != i ==> rank[i] < rank[parent[i]]` for every node i in [0, n). Requirements: Implement `size_t uf_find(UnionFind *uf, size_t i)` which (1) returns the representative root of i, and (2) compresses the path by re-pointing every node along the i->root chain directly to the root. Algorithm: walk parent pointers from i upward to the root r (parent[r] == r), then iterate again to overwrite parent[k] := r for every k on the path. Functional Correctness: the result must equal the pure logic value `spec_find{Pre}(uf, i)` defined axiomatically in the preamble (base case `parent[i]==i ==> spec_find=i`; step case `parent[i]!=i & rank[i]<rank[parent[i]] ==> spec_find(uf,i)==spec_find(uf,parent[i])`). After compression, `spec_find(uf, j) == spec_find{Pre}(uf, j)` must hold for every j in [0, n) — the logical partition is unchanged. Verification Challenges: (1) Logical Stability across mutation: WP must prove that overwriting parent pointers preserves spec_find at every index, even those off the compressed path; the rank array is unchanged (`uf->rank[k] == \at(uf->rank[k], Pre)`) which is the lever needed by the step axiom. (2) Termination: the upward walk decreases (n - rank[current]) since rank strictly increases along parent edges; supply `loop variant uf->n - uf->rank[cur];` or use a bounded iteration count. (3) Validity preservation: after compression, every reassigned parent pointer still satisfies `rank[k] < rank[parent[k]]` because the new parent is the root, whose rank is the maximum on the path. ACSL-specific notes: (a) The contract uses `assigns uf->parent[0 .. uf->n - 1];` — only parent is mutated; rank and n are framed by post-conditions. (b) The axiomatic block `UnionFindFind` in the preamble declares spec_find as a logic function with explicit `reads` clauses; do not introduce auxiliary axioms or top-level lemmas — the two given axioms (`spec_find_root`, `spec_find_step`) are sufficient when triggered by the loop invariant. (c) Use `-wp-model Typed`; pointer separation between `uf->parent` and `uf->rank` should be added as `requires \separated(uf->parent + (0..uf->n-1), uf->rank + (0..uf->n-1));` if needed for framing rank reads against parent writes. (d) The recursion in spec_find is inherently well-founded by rank; WP discharges this provided the invariant carries `rank[cur] < rank[parent[cur]]` along the walk.
