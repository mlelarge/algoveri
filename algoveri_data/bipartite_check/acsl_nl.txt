Your task is to implement a 2-Coloring (bipartiteness) check on an undirected graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is a CSR-encoded `Graph` (struct with `n`, `row_ptr` of length `n+1`, `col_idx` of length `row_ptr[n]`) satisfying `well_formed(g)`. The output buffer `colors` has length `g->n` and is writable; it must be `\separated` from the graph CSR arrays so that writes to `colors` cannot perturb the graph data. Requirements: Implement `check_bipartite(const Graph *g, int *colors)` returning `int`: `1` on success (filling `colors` with a valid 2-coloring) or `0` on failure. (1) Algorithm: Run BFS/DFS from every unvisited node, assigning each newly discovered node the opposite color of its parent. If any edge is found connecting two same-colored nodes, return `0`. Since the graph may be disconnected, repeat across components. (2) Functional Correctness: The postconditions are `\result == 1 ==> is_valid_coloring(g, colors)` and `\result == 0 ==> !is_bipartite(g)`, where `is_valid_coloring` requires `colors[v] in {0,1}` and `\forall u, v; has_edge(g, u, v) ==> colors[u] != colors[v]`, and `is_bipartite(g)` is the existential `\exists int *colors; is_valid_coloring(g, colors)`. Verification Challenges: (1) Coloring validity (positive case) is straightforward: maintain the invariant that the coloring of every visited node is consistent with its parent. (2) Non-bipartiteness (negative case) is the hardest part: you must prove that the conflict found at some edge `(u, v)` with `colors[u] == colors[v]` implies that no valid 2-coloring exists. The standard argument is a parity proof: tree-distance(start, u) + tree-distance(start, v) + 1 forms an odd cycle through `(u, v)`; in any 2-coloring `colors'[u] == colors'[v]` along that cycle, contradicting the edge `(u, v)`. (3) Connectivity: handle every component independently. ACSL-specific notes: (a) An additional `requires \separated(colors + (0..g->n-1), g)` plus separation against `g->row_ptr` and `g->col_idx` was added so that updates to `colors` cannot alias the graph CSR — Verus's `&Graph` borrow combined with a fresh `Vec<bool>` provides this implicitly. (b) The `is_bipartite` predicate is an existential over an arbitrary `int *colors`; in ACSL this is the standard idiom for "there exists some 2-coloring". The negative postcondition `!is_bipartite(g)` therefore reads "for every pointer `colors'`, `is_valid_coloring(g, colors')` fails". You may need ghost code that, given the conflict witness `(u,v)`, constructs the parity argument as an `assert` chain. (c) `assigns colors[0 .. g->n - 1];` is the only side effect — internal queues/visited arrays are local. (d) Supply `loop variant` for outer (component) and inner (BFS/DFS) loops; the natural variant is `g->n - visited_count`. (e) The default `-wp-model Typed` memory model suffices given the new `\separated` clauses. (f) Per project rules, no top-level lemmas or axioms in the user solution — encode the parity argument with in-function `assert` annotations and ghost code.
