Your task is to implement a linear-scan Bracket Matching algorithm and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input is a byte sequence `s` of length `n` (an `int`) with `0 <= n <= 1,000,000`; if `n > 0` then `s + (0..n-1)` must be readable. Requirements: Implement the function `bracket_match` returning an `int` with value `0` (false) or `1` (true). (1) Algorithm: Maintain a running balance counter `c` initialized to `0`. Iterate through the string: increment `c` on `'('` (ASCII 40), decrement `c` on `')'` (ASCII 41), leave `c` unchanged for other characters. If `c` becomes negative at any point, return `0` immediately. After the loop, return `1` iff `c == 0`. (2) Functional Correctness: The postcondition is `\result == 1 <==> is_matched(s, n)`, where `is_matched` is defined in the preamble as `total_weight(s, n) == 0 && valid_prefix_weights(s, n)`. The preamble provides: a logic function `char_weight` (1 for `'('`, -1 for `')'`, 0 otherwise), an axiomatic block `TotalWeight` defining `total_weight(s, n)` recursively (base `total_weight(s, 0) == 0`, step `total_weight(s, n) == total_weight(s, n-1) + char_weight(s[n-1])`), and the predicate `valid_prefix_weights(s, n)` asserting every prefix has non-negative weight. Verification Challenges: (1) Loop invariant: At each iteration index `i`, `c == total_weight(s, i)` and `\forall integer j; 0 <= j <= i ==> total_weight(s, j) >= 0`. Maintaining the prefix-non-negativity invariant requires showing the new prefix is non-negative iff `c >= 0` at that step. (2) Early-exit soundness: When returning `0` because `c < 0`, you must show that the prefix property is violated, hence `is_matched` is false. (3) Final equivalence: When the loop completes, `c == total_weight(s, n)`, so `c == 0` is equivalent to `total_weight(s, n) == 0`. ACSL-specific notes: (a) The contract is `assigns \nothing;` since the function is pure. (b) The axiomatic block `TotalWeight` is your only handle on the recursive sum — supply a `loop variant n - i;` for termination, and rely on the `total_weight_step` axiom to advance the invariant by one character. Frama-C/WP's Alt-Ergo backend usually instantiates this axiom automatically when the loop invariant references `total_weight(s, i)`. (c) `char_weight` is a non-recursive logic function so it inlines cleanly; you do not need to add a termination measure. (d) The default `-wp-model Typed` memory model is sufficient. (e) Cast the `const unsigned char *` correctly when calling predicates: write `is_matched((unsigned char *)s, n)` to match the predicate's parameter type. (f) Per project rules, do not introduce top-level lemmas or axioms in the user solution — extend reasoning only via in-function `assert`, `loop invariant`, and `loop variant` annotations.
