Your task is to implement the `delete` operation for a Ternary Search Tree (TST) and verify its correctness in ACSL using Frama-C/WP. A TST node carries a single character `val`, a Boolean `is_end` flag marking whether a key terminates here, and three children: `left` (BST-style for characters less than `val`), `right` (BST-style for characters greater than `val`), and `mid` (Trie-style for the next character of a matching key). Signature: `TstNode *tst_delete(TstNode *t, const unsigned char *key, size_t len)`; the function returns a (possibly null) tree from which `key` has been removed and any resulting "useless" nodes have been pruned. The deletion algorithm: descend along the search path comparing `key[0]` against `val`, recursing into `left`/`right` for unequal characters and into `mid` for equal characters with `key + 1, len - 1`; at the terminating node clear `is_end`; on the return path apply eager pruning — a node with `!is_end && left == \null && mid == \null && right == \null` is "empty" (`tst_is_empty`) and must be returned as `\null`, freeing the parent's pointer slot. Preconditions: `opt_well_formed(t)` (either null or a recursively well-formed, prune-clean TST), `len >= 0`, and the key is valid for read when non-empty. The contract specifies `assigns \nothing` and three postconditions: (1) the result is `opt_well_formed`, (2) the deleted key is no longer present (`!opt_contains(\result, key, len)`), and (3) frame: every other key has unchanged membership (`\forall k, kl; !seq_eq(k, kl, key, len) ==> opt_contains(\result, k, kl) <==> opt_contains{Pre}(t, k, kl)`). Verification challenges: (1) Path-copying versus in-place — the `assigns \nothing` contract forces a functional implementation: descend, build new nodes for every node on the path, reuse pointers to untouched siblings. (2) Pruning correctness — proving that returning `\null` when `tst_is_empty` holds preserves the global membership predicate `tst_contains` exactly because an empty node contains no keys. (3) Frame postcondition — the universal `\forall k, kl; !seq_eq(...)` quantifies over arbitrary external character pointers; discharge requires structural induction over the tree and the case analysis built into the recursive `tst_contains` predicate. (4) The `well_formed` invariant has both BST-style ordering on `left`/`right` (every key under `left` starts with a character `< val`, similarly for `right`) and the no-empty-children pruning rule; both must be re-established at every internal node returned. ACSL-specific notes: (a) `assigns \nothing` matches a path-copying functional implementation; in-place implementations would need a different `assigns` clause. (b) `allocates` clauses for newly built nodes should be added by the implementor; Frama-C/WP's `Typed` model handles fresh allocations automatically. (c) Use default `-wp-model Typed`; the recursive predicates are read-only structurally-well-founded fixpoints. (d) No top-level lemmas may be added; lemma functions about `tst_contains` distributing over the three child cases are essentially required, discharged by structural induction. (e) `len` is `size_t`; the `key + 1, len - 1` recursion shrinks `len` by one each `mid` step, providing the termination measure. The contract has no translation defect; the difficulty is the universally-quantified frame postcondition over arbitrary keys.
