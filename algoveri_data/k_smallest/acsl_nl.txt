Your task is to implement a Quick-Select-style algorithm to find the k-th smallest element of an array of 32-bit signed integers and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: `a` is `int32_t *` of length `n` with `0 <= n <= 10000`, `\valid(a + (0 .. n - 1))`, and `0 <= k < n`. Requirements: Implement `int32_t quick_select(int32_t *a, size_t n, size_t k)` so that on return: (1) the array is a multiset-permutation of its input (`is_permutation{Pre, Post}(a, n)`), and (2) the returned value is the value that index `k` would hold if the array were fully sorted (`is_kth_smallest{Pre, Post}(a, n, k, \result)`). Algorithm: classic in-place Lomuto/Hoare partition that picks a pivot, partitions `a` into `< pivot` and `>= pivot` halves, and recurses (or iterates) on the half containing index `k`. Verification Challenges: (1) Permutation Preservation: each swap preserves `count_occ` for every value, so the pre/post multiset equality follows from a sequence of two-cell swaps; encode via the inline `count_occ` axioms. (2) Selection Correctness: The k-th smallest must be witnessed by some sorted permutation of the original array having `\result` at index `k`; this is purely existential, so rather than fully sort the array (quick-select does not), the proof maintains a partition invariant: at the end, `a[0 .. k]` are all `<= \result`, `a[k] == \result`, and `a[k + 1 .. n)` are all `>= \result`. Combined with the permutation post, this is sufficient to construct (in a ghost variable) a fully sorted permutation in which index `k` holds `\result`. (3) Recursion/Iteration Bounds: provide a loop variant — the size of the "active window" decreases each iteration. ACSL-specific notes: (a) The `is_kth_smallest` predicate in the preamble is intended to express "there is *some* sorted permutation of the original array whose `k`-th cell equals `val`". A loose translation that requires `is_sorted{L2}(a, n)` would be too strong (quick-select does not sort the whole array). The correct ACSL form pins L1 to the input state and existentially quantifies a sorted-permutation witness — since ACSL cannot quantify over abstract sequences, we use a ghost buffer: `\exists int32_t *b; is_sorted_buffer(b, n) && same_multiset_buf{L1}(a, b, n) && b[k] == val`. (Concretely, the partition invariant produced by the algorithm constructs this `b` by sorting the lower half, placing the pivot, and sorting the upper half in ghost code.) (b) `count_occ` is a recursive logic function that the WP unfolds for swaps. (c) `assigns a[0 .. n - 1]` covers the in-place rearrangement. (d) Use a loop variant tied to the active window length. (e) Default `-wp-model Typed` plus Alt-Ergo/Z3 should suffice once you carry the partition invariant and the ghost sorted-witness; selection correctness without ghost code is intractable for SMT. (f) Do not introduce top-level lemmas; rely on inline ghost code and the preamble's `count_occ` and `is_sorted` predicates.
