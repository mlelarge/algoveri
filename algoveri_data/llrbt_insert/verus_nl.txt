Implement and verify the insertion algorithm for a Left-Leaning Red-Black Tree (LLRBT). The algorithm performs a standard recursive BST insertion (adding new nodes as Red) and then, during the return phase (unwinding the recursion), applies the three fixup operations (rotate_left, rotate_right, flip_colors) in a specific order to restore the tree invariants. **Verification Specification** Preconditions: The input tree is a valid LLRBT (satisfies BST ordering, black height balance, and color invariants). Postconditions: (1) Functional Correctness: The resulting view() is the union of the original set and the inserted value v. (2) Invariant Maintenance: The resulting tree must satisfy is_llrbt(). This implies: 1. BST Order: Preserved. 2. Black Height: Preserved (no paths have different black node counts). 3. Color Properties: No Red node has a Red child; no Red node is a right child. **Note:** The root of the result may be Red; it is the caller's responsibility to paint it Black.
