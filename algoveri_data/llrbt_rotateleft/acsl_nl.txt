Your task is to implement and verify a left rotation that enforces the "Left-Leaning" invariant of a Left-Leaning Red-Black Tree (LLRBT) in ACSL using Frama-C/WP. Preconditions: `t` is a non-null `LlrbtNode *` satisfying `is_bst(t)`, with a non-null `t->right` that is Red. Requirements: Implement `llrbt_rotateleft` returning an `LlrbtNode *`. (1) Algorithm: The classical left rotation - the original right child becomes the new root, the original root becomes the new left child, and the new root inherits the colour of the old root while the new left child becomes Red. Internal pointer rewiring: new_root = t->right; t->right = new_root->left; new_root->left = t; new_root->is_red = t->is_red; t->is_red = true. (2) Functional Correctness: The contract requires `\result` non-null, `is_bst(\result)`, set preservation (`in_view(\result, x) <==> in_view{Pre}(t, x)`), a non-null left child, `black_height(\result) == \at(black_height(t), Pre)`, `\result->is_red == \at(t->is_red, Pre)`, and `\result->left->is_red`. STRUCTURAL DEFECT (flagged challenge): The contract pairs `assigns \nothing;` with structural mutation: `\result->left == \at(t, Pre)` (the old root must become the left child of the new root) and `\result->is_red == \at(t->is_red, Pre)` while `\result != t` (the old right child is now the root). These are jointly satisfiable only if we either (a) heap-allocate a fresh node and rebuild the structure (incompatible with `assigns \nothing;` since `\fresh` allocation is technically a write to the global heap state), or (b) perform in-place pointer rewiring (incompatible with `assigns \nothing;` because we mutate fields). As written the spec is unsatisfiable in WP without admitting `\nothing` for fresh allocations - a known weakness of the standard `assigns \nothing` interpretation. The recommended fix is `assigns t->left, t->right, t->right->left, t->is_red, t->right->is_red;` with `ensures \result == \at(t->right, Pre);`. Verification Challenges: (1) Structural Defect (above) - blocks any sound in-place implementation. (2) BST Preservation: A left rotation preserves BST order because `t->right->val > t->val` and the in-order traversal is invariant. Proving this requires unfolding `is_bst` recursively and instantiating the membership predicate `in_view` with the new tree shape. (3) Black-Height Preservation: Critically, the rotation moves the Red right link to a Red left link without changing the count of Black nodes on any path. The proof relies on the fact that `t->right` was Red (by precondition), so its black-height equals the black-heights of its children; after rotation, the new root is Red iff the old root was, and the structural rearrangement keeps every path Black-count constant. The axiom `black_height_def` must be carefully instantiated. ACSL-specific notes: (a) The `assigns \nothing;` clause is the load-bearing defect; the simplest fix is to weaken to in-place writes and add `ensures \result == \at(t->right, Pre);`. (b) Logical predicates `is_bst` and `black_height` use one-level `reads` clauses; recursive lemmas tying these to subtree shape are required for the rotation argument. (c) `in_view` is a recursive predicate without a depth bound, which means WP's automatic unfolding stops at one level - manual ghost lemmas relating `in_view(\result, x)` to `in_view(t, x)` via the new structure are essential. (d) `-wp-model Typed` is sufficient. (e) No top-level lemmas required by contract, but `lemma left_rotation_preserves_bh` is a near-mandatory auxiliary. (f) Aliasing: in valid (acyclic) trees the various pointers do not alias, but ACSL has no built-in tree-acyclicity predicate; consider adding ghost preconditions of the form `\separated(t, t->right, t->right->left)` to enable WP to discharge the in-place rewiring.
