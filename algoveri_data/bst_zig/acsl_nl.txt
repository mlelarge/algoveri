Your task is to implement the Zig (single right rotation) operation on a Binary Search Tree and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input pointer `t` to a `BstNode` is valid for reading, satisfies `is_bst(t)`, and has a non-null left child (`t->left != \null`). Requirements: Implement `bst_zig(BstNode *t)` returning a non-null `BstNode *` whose root value equals the original `t->left->val` (the left child becomes the new root) and whose right subtree's root value equals the original `t->val`. Concretely, given the picture `t = (P, L = (Q, A, B), C)` (where `A`, `B`, `C` are subtrees, `P = t->val`, `Q = t->left->val`), the result is `(Q, A, (P, B, C))`. (1) Algorithm: Read `L = t->left`, attach `L->right` (subtree `B`) as the new `t->left`, then make `t` the right child of `L`, and return `L`. With `assigns \nothing` you instead allocate fresh nodes for the new root and the new `(P, B, C)` node. (2) Functional Correctness: Five postconditions: `\result != \null`, `is_bst(\result)`, view-equality `\forall integer x; in_view(\result, x) <==> in_view{Pre}(t, x)`, `\result->val == \at(t->left->val, Pre)`, and `\result->right != \null && \result->right->val == \at(t->val, Pre)`. The preamble provides `in_view` and `is_bst` (recursive predicates). Verification Challenges: (1) BST invariant preservation: in the rotated tree, the new root has value `Q` (originally the left-child's value), with `A` on its left and `(P, B, C)` on its right. The hard part is showing that every value in `B` is `> Q` (which holds because `B` was the right subtree of `L` originally, hence all values `> Q`) and `< P` (which holds because `B` was inside the left subtree of `t`, hence all values `< P`). Symmetrically, every value in `C` is `> P`, hence `> Q`. (2) Set Preservation: the multiset/view of the tree is the disjoint union of the values of the original three sub-pieces (`Q`, subtree `A`, subtree `B`, `P`, subtree `C`), and the rotation merely re-parenthesizes them. ACSL-specific notes: (a) The contract is `assigns \nothing;` — see the discussion in `bst_insert.acsl_nl.txt`. The cleanest implementation either (i) allocates fresh nodes for `L` and `t` and returns the fresh root, leaving the original tree intact, or (ii) does a more aggressive in-place mutation, which would force you to relax `assigns \nothing` to `assigns t->left, t->right, t->left->right, ...`. The spec is written for option (i). (b) `\at(t->left->val, Pre)` and `\at(t->val, Pre)` capture the *original* values, before any pointer manipulation. (c) `is_bst` is recursive; supply `decreases` on each call site or invoke it on subtrees of bounded ghost-height. (d) The default `-wp-model Typed` memory model is sufficient. (e) The view-equality postcondition `in_view(\result, x) <==> in_view{Pre}(t, x)` is the cleanest place to trigger Alt-Ergo's congruence reasoning: state `assert (\forall integer x; in_view(\result, x) <==> (x == Q || x == P || in_view{Pre}(A_root, x) || in_view{Pre}(B_root, x) || in_view{Pre}(C_root, x)));` and the symmetric for `t`, then chain. (f) Per project rules, no top-level lemmas or axioms in the user solution.
