Your task is to implement the Push-Relabel algorithm for maximum flow in a directed simple graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the input is a CSR-encoded `CapacityGraph *g` with n nodes (n <= 1000), `well_formed(g)` (valid row_ptr/col_idx, simple graph, capacities `\valid_read`), `capacities_bounded` (0 <= capacity <= 100,000), and source/sink indices s, t with 0 <= s, t < g->n and s != t. The flow buffer `f` is a flat row-major n*n int64_t matrix with `\valid(f + (0 .. n*n - 1))`. Requirements: implement `max_flow_value(g, s, t, f)` returning the max-flow value and writing a maximum flow assignment into f[u*n + v]. (1) Algorithm: initialize a preflow by saturating every edge out of s (`f[s*n + v] = capacity(s,v)`), set heights h[s] = n and h[v] = 0 for all other v, and maintain an excess[v] array. Repeatedly choose an active node (excess[u] > 0, u != s, t) and either Push along an admissible edge (h[u] == h[v] + 1, residual capacity > 0) or Relabel (raise h[u] to one more than the minimum height of residual neighbors). Terminate when no active nodes remain. (2) Functional Correctness: prove `is_max_flow(g, f, s, t)` (which unfolds to `respects_capacity`, `is_conserved` away from {s, t}, non-negativity, and global optimality against every other valid flow) and `\result == flow_val(g, f, s) == total_flow_out(g, f, s) - total_flow_in(g, f, s)`. Verification Challenges: (1) Height Invariant — for every edge (u, v) with residual capacity, `h[u] <= h[v] + 1`; this is the loop-invariant heart of push-relabel and prevents infinite cycling. (2) Preflow-to-Flow — termination requires that no active nodes remain, which converts the preflow (nodes can have excess) into a true flow (excess == 0 everywhere except s and t). The proof has to chain from "no active node" to "is_conserved holds." (3) Termination via potential function — the standard argument bounds total work by O(n^2 * m) using Φ = sum of heights of active nodes; encode Φ as a ghost variable and supply it as the `loop variant`. (4) Optimality — `is_max_flow`'s global quantifier over arbitrary FlowMaps is research-grade; the textbook proof goes through the max-flow/min-cut theorem (the heights induce a cut whose capacity equals the flow value when no augmenting path exists). Expect to encode a ghost cut and prove flow_val == capacity_of_cut at termination. ACSL-specific notes: (a) The contract requires `\separated(f + (0 .. n*n - 1), g, g->row_ptr + (0 .. n), g->col_idx + (..), g->col_weight + (..))` so writes to f do not invalidate the `has_capacity` reads. (b) Use the height-potential Φ as the `loop variant` for the main outer loop; per-operation termination of Push/Relabel needs its own variant (residual capacity, h[u] respectively). (c) The `sum_flow_out_recursive` / `sum_flow_in_recursive` axiomatic blocks are the only logic state — do not add free-standing axioms; cut-induction lemmas belong inline as ghost code. (d) `-wp-model Typed` is sufficient; expect that the cut-based optimality argument is the dominant manual proof effort and may be left as an explicit ghost lemma boundary if a complete proof exceeds Frama-C/WP's tractable scope.
