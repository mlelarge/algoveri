Your task is to implement a Naive String Search algorithm to find all occurrences of a needle (pattern) within a haystack (text) and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input text and pattern are sequences of bytes (unsigned char) and their lengths are at most 1,000,000 to ensure index arithmetic does not overflow. The output buffer `out` is sized for at most `hay_len + 1` indices (the safe upper bound when needle_len == 0); `out_len` is a single integer. Requirements: Implement the function `naive_search` which writes matching start indices into `out[0 .. *out_len - 1]` and stores the count in `*out_len`. (1) The function must iterate through every possible starting position in the haystack. (2) For each position, it must perform a character-by-character comparison to check for a match. (3) Soundness & Completeness: You must prove that every index returned is a valid match (predicate `matches_at(hay, hay_len, needle, needle_len, out[i])` for all i in [0, *out_len)) and that no valid match is omitted (every match index appears somewhere in `out[0 .. *out_len - 1]`). Verification Challenges: (1) Nested Loop Invariants: You must correctly specify the invariant for the inner comparison loop to prove it correctly identifies a match or mismatch. (2) Quantifier Instantiation: Proving completeness involves complex quantifiers (forall indices, if a match exists, it is in the result), which may require manual triggers or ghost loops to show the iteration covers the entire search space. ACSL-specific notes: (a) The contract requires `\separated(out_len, out + (0 .. hay_len))` so that the final `*out_len = count;` write does not alias an `out[k]` cell — without this the soundness/completeness postconditions cannot be discharged. (Verus's borrow-checker provides this guarantee implicitly via `&mut` references; in ACSL it must be stated.) (b) The `unsigned char` element type means each character lives in [0, 255]; pointer-to-character casts in the predicate use `(unsigned char *)` so `\valid_read` ranges align with the predicate's reads. (c) The default `Typed` memory model is sufficient with the `\separated` clause in place.
