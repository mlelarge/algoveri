Your task is to implement an algorithm computing the length of the Longest Strictly Increasing Subsequence (LIS) and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: You are given a `const int32_t *` array `a` of length `n` with `0 <= n <= 3000`; when `n > 0`, the buffer must be read-valid over `0 .. n - 1`. Requirements: Implement `longest_increasing_subsequence` returning an `int` equal to the LIS length. (1) Algorithm: The simplest verifiable approach is the `O(n^2)` DP where `lis[i]` is the length of the longest strictly increasing subsequence ending at index `i`; the result is the maximum over all `i`. The `O(n log n)` patience-sorting variant is correct but harder to verify. (2) Functional Correctness: Subsequences are represented as index arrays `int *idx` of length `k`, validated by the predicate `is_valid_is(seq, n, idx, k)`: indices strictly increasing, in range `[0, n)`, and corresponding values strictly increasing. You must prove Upper Bound: every valid increasing subsequence with `k > 0` has `k <= \result`; Existence: some valid increasing subsequence has length exactly `\result`; and the trivial bound `0 <= \result <= n`. Verification Challenges: (1) Existence Witness: The `\exists int *idx, integer k; ...` postcondition requires materialising a witness index array - typically a ghost array filled by traceback over `prev[i]` pointers. (2) Upper Bound by Induction on the Subsequence's Last Element: The classical proof inducts over the last index of an arbitrary valid subsequence; at each step the inductive hypothesis is `lis[idx[k-1]] >= k`, which is exactly the loop invariant of the inner DP loop. (3) Quantifier Instantiation: The predicate `is_valid_is` carries three nested universal quantifiers over arbitrary index arrays; SMT solvers need triggers via `idx[i]` and `seq[idx[i]]` to instantiate them. ACSL-specific notes: (a) The contract is `assigns \nothing;` because the function is pure - the `lis` and (optional) `prev` DP arrays live on the stack. (b) No `\separated` clause is needed for the read-only input `a`. (c) `idx` in the spec is typed `int *` (not `size_t *`) to match the Verus `Seq<int>`; this is fine because the bounds in `is_valid_is` constrain `0 <= idx[i] < n` independently. (d) `-wp-model Typed` is sufficient. (e) No top-level lemmas are required; the predicate `is_valid_is` is non-axiomatic (definitional), so the existence witness can be expressed entirely in ghost code without auxiliary axioms. (f) The bound `n <= 3000` keeps `lis[i]` and `\result` safely inside `int` range.
