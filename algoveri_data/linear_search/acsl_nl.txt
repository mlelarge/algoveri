Your task is to implement the lower-bound search (linear scan variant) and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input is a sorted array `seq` of `int32_t` with length `n`, where `0 <= n <= 0x7FFFFFFF`, and `seq[0..n-1]` is read-valid. The array satisfies the predicate `is_sorted(seq, n)` defined in the preamble as the standard non-strict ordering: for all `0 <= i <= j < n`, `seq[i] <= seq[j]`. Requirements: Implement the function `linear_search_lower_bound` returning a `size_t`. (1) Algorithm: Walk left-to-right with index `i`, advancing as long as `seq[i] < target`; return the first index `i` where `seq[i] >= target`, or `n` if no such index exists. While the asymptotic complexity is `O(N)`, the proof obligations are identical to those of binary search's lower-bound variant. (2) Functional Correctness: You must prove the boundary postconditions `\forall integer i; 0 <= i < \result ==> seq[i] < target` (everything to the left is strictly less) and `\forall integer i; \result <= i < n ==> seq[i] >= target` (everything to the right is at least the target). Verification Challenges: (1) Two-Sided Loop Invariant: The single scan needs an invariant carrying both the left-side predicate (already-passed elements are `< target`) and termination structure (`0 <= i <= n`). Once the loop exits, the right-side predicate is established by combining the loop-exit condition (`seq[i] >= target` or `i == n`) with the sortedness hypothesis: any `j` to the right is `>= seq[i] >= target`. (2) Sortedness Instantiation: WP needs to instantiate `is_sorted` with the right pair `(i, j)` to derive the right-side postcondition; an explicit `assert` after the loop or a careful invariant phrasing usually suffices. ACSL-specific notes: (a) `n` is `size_t` so non-negative by C typing; the bound `n <= 0x7FFFFFFF` keeps `i` arithmetic safely inside `int` range, but the loop counter should still be `size_t` to mirror the parameter type. (b) The contract uses `assigns \nothing;` since the function is pure. (c) No `\separated` clause is required because all reads are from a single `const`-qualified buffer. (d) The default `-wp-model Typed` memory model is sufficient. (e) The axiomatic preamble contains only `is_sorted`; no top-level lemmas are needed - the SMT backends (Alt-Ergo, Z3) discharge both invariant preservation and the final boundary properties directly from the sortedness hypothesis with a well-phrased loop invariant.
