Your task is to implement the Rod Cutting algorithm (CLRS dynamic programming) and verify its functional correctness in ACSL using Frama-C/WP. Given a rod of length `n` and a price table `prices[0 .. m-1]` where `prices[i]` is the price of a piece of length `i+1`, return the maximum revenue obtainable by cutting the rod into pieces and summing their prices. Signature: `uint64_t rod_cutting(size_t n, const uint64_t *prices, size_t m)`. Preconditions: `0 <= n <= 1000`, `n <= m`, `prices` valid for read over `[0, m-1]` when `m > 0`, and every entry bounded: `prices[i] <= 10000`. The contract specifies `assigns \nothing` and two postconditions stated over an axiomatic preamble: (1) Upper bound — for every valid strategy `(cuts, k)` (a list of positive piece lengths summing to `n`, with each piece referencing a valid index of `prices`), `calculate_revenue(cuts, k, prices, m) <= \result`; (2) Existence — there exists such a `(cuts, k)` whose revenue equals `\result`. The preamble defines `sum_lengths` and `calculate_revenue` as `logic` functions with axioms for the empty and inductive cases, and `is_valid_strategy` as a predicate. The natural implementation uses bottom-up DP: `dp[j] = max over 1 <= p <= j of (prices[p-1] + dp[j - p])` for `j` in `[1, n]`, then return `dp[n]`. Verification challenges: (1) DP-array invariant — at the outer-loop boundary, `dp[j]` must equal the supremum over all strategies for length `j`; expressing this requires a quantifier over arbitrary cuts-arrays. (2) Existence postcondition — to prove the existential you typically maintain a parallel "witness" ghost array `choice[j]` that records the optimal first cut for length `j`, then reconstruct an explicit witness `cuts[]`. This means allocating ghost storage. (3) Upper-bound proof — the inductive step needs `calculate_revenue(p :: rest) = prices[p-1] + calculate_revenue(rest)` and `sum_lengths(p :: rest) = p + sum_lengths(rest)`; both follow directly from the given axioms but require explicit triggers. (4) Overflow — with `n <= 1000` and `prices[i] <= 10000`, the maximum revenue fits in `uint64_t` (bounded by `1000 * 10000 = 10^7`); WP's `Typed` model verifies this without `-warn-unsigned-overflow` issues. ACSL-specific notes: (a) The `RodCutting` axiomatic provides `sum_lengths` and `calculate_revenue` over arbitrary `int *cuts` pointers; postconditions need concrete ghost-array witnesses for the existential. (b) `assigns \nothing` is correct because writes are confined to local stack arrays. (c) Use default `-wp-model Typed`; no `\separated` needed. (d) This problem is SMT-hard, not a translation defect: combining the universal upper-bound with the existential witness is a fundamental DP verification challenge. (e) No top-level lemmas may be added, but ghost lemma functions for `sum_lengths`/`calculate_revenue` distributivity over `cuts ++ [p]` are essentially required.
