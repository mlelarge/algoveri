Your task is to implement the Search operation for a Byte-based Trie (Prefix Tree) and verify its functional correctness in ACSL using Frama-C/WP. The trie node is `typedef struct TrieNode { bool is_end; struct TrieNode *children[256]; } TrieNode`, mirroring the Verus `Node { is_end: bool, children: Vec<Option<Box<Node>>> }` with the children vector forced to length 256. Requirements: Implement `bool trie_search(const TrieNode *t, const unsigned char *key, size_t len)` that returns true iff the byte sequence key[0..len-1] is present in the abstract set of keys stored in the tree. (1) Algorithm: starting from the root, for each byte b = key[i] consult `t->children[b]`; if it is NULL return false; otherwise descend. After consuming all len bytes return `t->is_end`. The empty-tree case (t == NULL) must return false directly. (2) Functional Correctness: the result must match the predicate `opt_contains(t, key, len) <==> (t != NULL && trie_contains(t, key, len))` from the preamble, where `trie_contains` recurses on `t->children[key[0]]` and `key + 1` until len reaches 0, at which point it asks for `t->is_end`. The well-formedness invariant `opt_well_formed` (children array size 256, no useless nodes) is given as a precondition. Verification Challenges: (1) Recursive predicate over a heap-allocated tree: WP must unfold `trie_contains` along the descent loop. The natural implementation is iterative: maintain a running pointer `cur` and an index `i`, with loop invariant `(\result correctly so-far) <==> trie_contains{Pre}(\at(t,Pre), key, len) && cur points to the subtree corresponding to the i-byte prefix consumed`. (2) Termination: variant is `len - i`. (3) Pointer aliasing: as a pure read-only function, no `\separated` clauses are needed beyond the implicit framing; `assigns \nothing;`. ACSL-specific notes: (a) The well_formed predicate is recursively defined and WP unfolds it lazily; you may need an axiomatic helper or `lemma` proving "well_formed(t) ==> for any byte c, t->children[c] is either NULL or well_formed". This is best stated via a ghost lemma rather than a top-level axiom. (b) The predicate `trie_contains` mixes `\valid_read` of the children pointer chain with the recursive structural recursion — WP's heap reasoning under `-wp-model Typed` handles this provided every child pointer dereference along the path was implicitly validated by `well_formed`. (c) `bool` here is the C99 stdbool; ACSL converts true/false via the standard predicate. (d) No top-level lemmas should be required — the recursive predicates plus loop invariants suffice; in particular, do not introduce axioms for `trie_contains`, since it is already a recursive predicate definition. (e) Empty key (len == 0): the contract evaluates `trie_contains(t, key, 0) == t->is_end`, which the implementation must handle as a special case that returns `t->is_end` after the loop body has not executed.
