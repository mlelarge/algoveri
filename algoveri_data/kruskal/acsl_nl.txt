Your task is to implement Kruskal's Algorithm to find the Minimum Spanning Tree (MST) of a connected, undirected, weighted simple graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is provided in CSR (compressed sparse row) form via `WeightedGraph` with fields `n` (node count, up to 100,000), `row_ptr` (length `n+1`), `col_idx` and `col_weight` (length `row_ptr[n]`). The predicates `well_formed(g)` (CSR validity, simple-graph constraint), `weights_bounded(g)` (`-100,000 <= w <= 100,000`), and `graph_is_connected(g)` are all required. Caller-allocated output buffers `out_u`, `out_v`, `out_w` each have capacity `n - 1`, and the three are pairwise `\separated`. Requirements: Implement `kruskal_mst` returning `size_t` (= `n - 1` on success). (1) Algorithm: Materialise the edge list from the CSR (avoid duplicating undirected edges by keeping `u < v`), sort by weight, and iterate; use Union-Find to test whether each edge would close a cycle. Add edges that connect two different components until exactly `n - 1` are accumulated. (2) Functional Correctness: Prove `is_mst(g, out_u, out_v, out_w, \result)`, which expands to: the output edges are valid (`are_valid_edges`), they span the graph (`is_spanning` via `edges_connected`), `\result == g->n - 1`, and `total_weight(out_w, \result) <= total_weight(ow, olen)` for every other spanning tree of the graph. Verification Challenges: (1) Union-Find Soundness: You must verify that `find(u) != find(v)` characterises "edge `(u,v)` does not lie in the current spanning forest". This requires a ghost field per node tracking which component it lies in, plus invariants that the parent pointer chain reaches the canonical root. (2) Cut Property (Greedy Optimality): The classical proof - "the lightest edge crossing any cut belongs to some MST" - must be expressed as a ghost lemma; the loop invariant is "the current accumulated edge set is a subset of some MST". This is the hardest piece: reasoning over the universally quantified `\forall int *ou, int *ov, int64_t *ow, integer olen; is_spanning_tree(...) ==> ...` requires producing or transforming an arbitrary alternative spanning tree. (3) Path Existence Encoding: `edges_connected` and `nodes_have_path` use `\exists int *p, integer plen` where `p` is an arbitrary integer-valued path; you must materialise such witnesses both for the input connectivity (rarely needed in proof scaffolding) and the output spanning property. ACSL-specific notes: (a) The contract `assigns out_u[0..g->n-1], out_v[0..g->n-1], out_w[0..g->n-1];` requires the three buffers to be mutually `\separated` (now stated as preconditions) so that writes to one do not alias the others. (b) Internal Union-Find arrays are local; if you allocate the edge list or DSU on the heap you must add `\separated` clauses against `g->row_ptr`, `g->col_idx`, `g->col_weight`. (c) The `total_weight` axiom block is an inductive sum over `ew`; both the spec and implementation must agree on the orientation - the axioms recurse from the right (`elen-1` down). (d) `-wp-model Typed` is sufficient; the i64 weights fit in mathematical integers natively so no overflow reasoning is needed. (e) No top-level lemmas, but extensive ghost lemmas about Union-Find correctness and the cut property are required - the global optimality postcondition is at the frontier of what is currently feasible with WP without significant manual proof support.
