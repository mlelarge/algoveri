Your task is to implement the Longest Common Subsequence (LCS) algorithm and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: You are given two `const unsigned char *` byte arrays `s` and `t` of lengths `n` and `m` respectively, with `0 <= n <= 3000` and `0 <= m <= 3000`. Each array must be read-valid over `0 .. len - 1` when its length is positive (when length is zero, no validity is required). Requirements: Implement `solve_longest_common_subsequence` returning an `int` equal to the LCS length defined by the axiomatic predicate `lcs_spec`. (1) Algorithm: Use the standard 2-D DP `dp[i][j]` where `dp[i][j] = lcs_spec(s, i, t, j)`; iterate over `i` from 0 to `n` and `j` from 0 to `m`. The recursive case mirrors the spec exactly: if `s[i-1] == t[j-1]`, `dp[i][j] = 1 + dp[i-1][j-1]`; otherwise `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`. The space-optimised rolling-array version is also acceptable but adds invariant-rephrasing work. (2) Functional Correctness: The axiomatic block `LcsSpec` defines `lcs_spec(s, n, t, m)` recursively with three axioms: base case (`n <= 0 || m <= 0`), match case (`s[n-1] == t[m-1]`), and mismatch case (max of two subproblems). The single postcondition is `\result == lcs_spec(s, n, t, m)`. Verification Challenges: (1) Loop Invariant Mirroring the Recursive Spec: For each cell `dp[i][j]` you must establish `dp[i][j] == lcs_spec(s, i, t, j)`; this requires the loop invariants of both outer and inner loops to track all already-filled cells, and to instantiate the appropriate axiom (base/match/mismatch) at the correct concrete indices. (2) Axiom Triggering: The three `lcs_spec_*` axioms are universally quantified over arrays and lengths; SMT solvers need explicit hints (assertions naming `lcs_spec(s, i, t, j)` with the specific `i`, `j`) to fire the right axiom. (3) `max2` Reasoning: The mismatch case uses `max2(a, b) = (a > b) ? a : b`; while the conditional logic is trivial for SMT, combining it with the axiomatic recurrence under quantifiers can require auxiliary lemmas. ACSL-specific notes: (a) The DP table `dp` is a local `(n+1) x (m+1)` `int` array (or two rolling rows); since it is local, no `\separated` clause vs. `s`/`t` is required. (b) The contract is `assigns \nothing;` because the function is pure - all writes are to stack-local DP storage. (c) The axiomatic `reads s[0..n-1], t[0..m-1]` clause means `lcs_spec` is sensitive only to the byte contents within those ranges; this matches the C-level argument. (d) `-wp-model Typed` is sufficient. (e) No top-level lemmas are strictly required, but a monotonicity lemma `lcs_spec(s, n, t, m) <= min(n, m)` is convenient for bounding `\result` and for the loop invariant that all DP cells fit in `int`. (f) Strings are passed as `unsigned char *` (not signed `char *`) so the equality `s[n-1] == t[m-1]` matches both at the C level and inside the axiom.
