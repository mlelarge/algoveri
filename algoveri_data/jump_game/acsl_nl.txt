Your task is to implement the Jump Game greedy reachability algorithm and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: `nums` is `const uint64_t *` of length `n` with `0 < n <= 10000` and every `nums[i] <= 10000`; the predicate `valid_nums((uint64_t *)nums, n)` packages these together. The signature is `int can_jump(const uint64_t *nums, size_t n)`. Requirements: Return `1` iff there exists a strictly increasing sequence of indices `path[0] = 0, path[1], ..., path[plen-1] = n - 1` with each step bounded by `nums[path[i]]` (`path[i+1] <= path[i] + nums[path[i]]`). Algorithm: classic greedy — sweep `i = 0 .. n - 1` maintaining `max_reach`, return `0` if `i > max_reach`, otherwise update `max_reach = max(max_reach, i + nums[i])`; return `1` after the sweep. Verification Challenges: (1) Soundness: when you return `1`, exhibit a witness `int *path, integer plen` with `is_valid_jump_path(path, plen, nums, n)`. The natural witness is built greedy-style: from each visited `i`, jump to `i + nums[i]` (capped at `n - 1`) until you land on `n - 1`. Carry this construction in a ghost array. (2) Completeness: when you return `0`, no such path exists. The key invariant is `max_reach == max over all valid prefix paths of (path[k] + nums[path[k]])`; once `i > max_reach` no path of any length can extend past `i`. Encode this as `\forall path, plen; is_valid_jump_path-prefix(path, plen, ...) ==> path[plen - 1] <= max_reach`. (3) Termination: a single `for` loop with variant `n - i`. ACSL-specific notes: (a) `can_reach` is defined existentially via `\exists int *path, integer plen; is_valid_jump_path(path, plen, nums, n)`; under the default `Typed` model WP treats this as quantification over phantom buffers, so soundness must be demonstrated with a concrete (ghost) witness array allocated inline. (b) `is_valid_jump_path` requires `path[i + 1] > path[i]` and `path[i + 1] <= path[i] + nums[path[i]]`, plus `path[0] == 0`, `path[plen - 1] == n - 1`. The `>` (strict) constraint forbids zero-length jumps; this matches the Verus spec. (c) `assigns \nothing` (pure function). (d) `nums[i] + i` could overflow `uint64_t` only if `i + nums[i] > 2^64`, which the preconditions (`n <= 10000`, `nums[i] <= 10000`) preclude — but write the inner update in `integer` arithmetic inside the contract to be safe. (e) Use a loop variant `n - i` for termination. (f) Default `-wp-model Typed` plus Alt-Ergo/Z3 should suffice when you supply ghost witness/refutation code; SMT alone cannot find paths in a buffer of unknown length. (g) Do not introduce top-level lemmas; inline ghost assignments to a witness array suffice.
