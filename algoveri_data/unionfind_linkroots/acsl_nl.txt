Your task is to implement the link_roots operation for a Union-Find (Disjoint Set Union) data structure and verify its functional correctness in ACSL using Frama-C/WP. The structure is encoded as `typedef struct { size_t *parent; size_t *rank; size_t n; } UnionFind`, with `is_valid_uf` requiring the rank-strict-monotone DAG property `parent[i] != i ==> rank[i] < rank[parent[i]]` for every i in [0, n). Requirements: Implement `void uf_link_roots(UnionFind *uf, size_t root_i, size_t root_j)`, called only on two indices that are currently roots (parent[root_i] == root_i, parent[root_j] == root_j). The operation merges their trees using union-by-rank: (1) if rank[root_i] < rank[root_j], set parent[root_i] := root_j; (2) if rank[root_j] < rank[root_i], set parent[root_j] := root_i; (3) if equal, attach root_i under root_j and increment rank[root_j] by 1. Postconditions: `is_valid_uf` is preserved, n is preserved, `spec_find(uf, root_i) == spec_find(uf, root_j)` (the two roots now share a representative), and a frame property: every other index that was a root before is still a root afterward. Verification Challenges: (1) Rank invariant after an attach: when `rank[root_i] < rank[root_j]` and we set parent[root_i] := root_j, the strict-monotone property `rank[root_i] < rank[parent[root_i]] == rank[root_j]` must continue to hold; the equal-rank case bumps the new root's rank by one to preserve the invariant. (2) Frame for non-root vertices: the operation only writes `parent[root_i]` (and possibly `rank[root_j]`); for any other k, parent[k] and the predicate-relevant rank values are unchanged, so `spec_find(uf, k) == spec_find{Pre}(uf, k)` follows from the axioms. (3) `spec_find` over the merged structure: after the attach, spec_find(uf, root_i) follows the new edge to root_j, where parent[root_j] == root_j, so spec_find(uf, root_i) == root_j == spec_find(uf, root_j). ACSL-specific notes: (a) Crucial: the contract MUST contain `requires \separated(uf->parent + (0 .. uf->n - 1), uf->rank + (0 .. uf->n - 1));` — the link operation reads rank while writing parent, and without this clause WP cannot frame the rank reads against the parent updates. Verus's borrow checker provides this guarantee implicitly via `&mut self`, but in ACSL it must be stated explicitly. This was the primary translation defect from the source spec and has been added. (b) The clause `assigns uf->parent[0 .. uf->n - 1], uf->rank[0 .. uf->n - 1];` covers both arrays; only the rank cell of the chosen new root is actually changed in the equal case, but assigns is a may-write set. (c) The axiomatic block `UnionFindLink` provides the same `spec_find_root` / `spec_find_step` axioms used in `uf_find`; no top-level lemmas are needed. (d) Use `-wp-model Typed`; size_t arithmetic for the rank bump should be guarded against overflow but rank < n <= SIZE_MAX makes this trivial.
