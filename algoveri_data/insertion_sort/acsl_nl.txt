Your task is to implement Insertion Sort on an array of 32-bit signed integers and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: `a` is `int32_t *` of length `n` with `0 <= n <= 1000`; when `n > 0` the buffer must satisfy `\valid(a + (0 .. n - 1))`. Requirements: Implement `void insertion_sort(int32_t *a, size_t n)` so that on return `is_sorted(a, n)` and the multiset of values is preserved (`same_multiset{Pre, Post}(a, n)`). Algorithm: For `i` from `1` to `n - 1`, lift element `a[i]` and shift larger values to its right one step at a time until the correct insertion position is found within the already-sorted prefix `a[0 .. i - 1]`. Verification Challenges: (1) Outer-loop invariant: `is_sorted(a, i)` and `same_multiset{Pre, Here}(a, n)`, i.e. the prefix is sorted and the entire array is still a multiset-permutation of the input. (2) Inner-loop invariant: while shifting, the element being placed has been "lifted" into a temporary; the inner loop preserves the multiset and maintains a "hole" — express this by stating that the values in `a[0 .. j) ++ {key} ++ a[j + 1 .. i]` form the same multiset as `a[0 .. i]` at the start of the outer iteration, plus the order constraint on `a[0 .. j)` and `a[j + 1 .. i]`. (3) Termination: outer loop variant `n - i`, inner loop variant `j` (decreasing toward 0). ACSL-specific notes: (a) The preamble defines `is_sorted(a, n)` directly and a recursive `count_occ{L}(a, n, v)` plus `same_multiset{L1, L2}(a, n) == \forall v; count_occ{L1}(a, n, v) == count_occ{L2}(a, n, v)`; this captures permutation as multiset equality. (b) Use `same_multiset{Pre, Here}(a, n)` in loop invariants and lemmas on swaps: a single adjacent swap preserves `count_occ` for every value, which Frama-C/WP can usually discharge automatically once you prove the swap touches only two cells. (c) `assigns a[0 .. n - 1]` is correct since insertion sort writes within the buffer only. (d) Beware the empty-array case (`n == 0`): the requires uses `n == 0 || \valid(a + (0 .. n - 1))`, so the function must return without dereferencing `a`. (e) Loop variants are mandatory (Frama-C/WP demands them for termination of `while` loops). (f) Default `-wp-model Typed` plus Alt-Ergo/Z3 should suffice — `count_occ` is a recursive logic function with two equational rules, so any swap proof reduces to an integer arithmetic obligation. (g) Do not add top-level lemmas; carry per-iteration `count_occ` reasoning inline via ghost asserts where needed.
