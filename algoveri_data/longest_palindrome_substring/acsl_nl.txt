Your task is to implement Manacher's algorithm for the Longest Palindromic Substring and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the input s is a sequence of bytes (unsigned char) of length n with 0 <= n <= 1,000,000, so even the transformed string of length 2n+1 fits comfortably in size_t/int. The outputs out_start and out_len each point to a single int. Requirements: implement the function `longest_palindromic_substring` which writes the start index and length of a longest palindromic substring of s into *out_start and *out_len. (1) Algorithm: build the modified string T by inserting a unique separator (e.g., '#') between every character of s and at the outer boundaries so odd- and even-length palindromes are handled uniformly. Maintain an array P with P[i] equal to the radius of the palindrome centered at T[i], plus a current center C and right boundary R. For each index i, initialize P[i] using mirror symmetry around C (P[i] = min(R - i, P[2C - i]) when i < R, otherwise 0), expand greedily while T[i - 1 - P[i]] == T[i + 1 + P[i]], then update C and R if i + P[i] > R. Finally extract the maximum P[i] and map it back to (start, length) in s. (2) Functional Correctness: prove that valid_subrange(n, *out_start, *out_len) holds, that palindrome_range(s, n, *out_start, *out_len) holds (the predicate from the preamble: every i in [0, len) satisfies s[start + i] == s[start + len - 1 - i]), and that for every (i, len) with valid_subrange(n, i, len) and palindrome_range(s, n, i, len), len <= *out_len. Verification Challenges: (1) Mirror correctness — proving that P[2C - i] correctly bounds P[i] inside the current envelope requires reasoning about palindromic symmetry of T. (2) Maximality — proving that no center was missed and that the greedy expansion terminates with the true radius is the hardest universally-quantified obligation. (3) Mapping back — once a center c and radius r in T are fixed, deriving start = (c - r) / 2 and len = r in s is a small but error-prone integer-arithmetic step. ACSL-specific notes: (a) The contract requires `\separated(out_start, out_len, s + (0 .. n - 1))` so that writing *out_start does not alias *out_len or any input byte (Verus's `&mut` and `Seq` references give this for free). (b) Each loop (outer scan over i, inner expansion, final maximum scan) needs an explicit `loop variant` for termination — naturally `n - i`, `n - (i + 1 + P[i])`, etc. (c) The default `-wp-model Typed` is sufficient with `\separated` in place. (d) No top-level lemmas should be introduced; auxiliary facts about palindromic symmetry belong inside the function as ghost code, not as global axioms.
