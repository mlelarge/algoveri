Your task is to implement a basic Primality Test (Naive Trial Division) and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input n is a uint64_t. Requirements: Implement the function `check_prime` which returns 1 if n is prime and 0 otherwise. (1) Algorithm: Use a simple trial-division loop that walks every integer d from 2 up to n - 1 and checks whether d divides n. As soon as a divisor is found return 0; if the loop completes without finding one, return 1 when n > 1 and 0 when n <= 1. (2) Functional Correctness: You must verify that your result matches the mathematical definition encoded by the predicate `is_prime`, defined in the preamble as `n > 1 && (\forall integer d; 1 < d < n ==> !divides(d, n))`, where `divides(d, n) == (d != 0 && n % d == 0)`. Verification Challenges: (1) Loop Invariant: Maintain `\forall integer d; 1 < d < i ==> !divides(d, n)` for the current loop counter i; this captures "no divisor in [2, i) has been found so far". (2) Quantifier Instantiation: When the loop exits with i == n, you must combine the invariant with the loop termination condition to derive the universal `\forall integer d; 1 < d < n ==> !divides(d, n)`, which is exactly the second conjunct of `is_prime`. (3) Special cases: n == 0 and n == 1 must return 0; the contract requires `\result == 1 <==> is_prime(n)` so the implementation must reject these explicitly before the loop. ACSL-specific notes: (a) The integer/modulo operators on uint64_t are mapped to mathematical `%` in the predicate because both operands are non-negative; no signedness reasoning is needed. (b) The function is pure, so the contract uses `assigns \nothing;`. (c) Termination: supply a `loop variant n - i;` or equivalent strictly-decreasing quantity. (d) Use the default `-wp-model Typed` memory model — the function takes no pointers. (e) No top-level lemmas are required: the predicates `divides` and `is_prime` in the preamble are the only logical state needed, and Alt-Ergo/Z3 should discharge the post-loop quantifier elimination directly once the invariant is in place. Avoid introducing axioms beyond what the preamble provides; ghost variables to record the loop counter are unnecessary because the invariant itself is sufficient.
