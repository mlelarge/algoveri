Your task is to implement the Edmonds-Karp maximum-flow algorithm on a directed simple graph with non-negative integer capacities and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is a CSR-encoded `CapacityGraph` with `col_weight[e]` storing capacities; `well_formed(g)` enforces row-pointer monotonicity, in-range neighbours, and the simple-graph property (no two outgoing edges to the same target). `capacities_bounded(g)` constrains `n <= 1000` and every capacity to `[0, 100000]`. Source and sink satisfy `0 <= s < g->n`, `0 <= t < g->n`, `s != t`. The caller provides an `int64_t *f` of length `n * n` representing the flow as a row-major matrix. Requirements: Implement `int64_t max_flow_value(const CapacityGraph *g, size_t s, size_t t, int64_t *f)` which fills `f` with a maximum flow and returns `flow_val(g, f, s)`. The contract demands `is_max_flow(g, f, s, t)`, i.e. `f` is non-negative, respects capacity, conserves flow at all internal vertices, and dominates every other valid flow on `flow_val`. Algorithm: Repeatedly perform BFS on the residual graph (residual edge `(u -> v)` has capacity `cap(u, v) - f[u, v] + f[v, u]`), augment along the shortest residual path by the bottleneck capacity, and stop when no augmenting path remains. Verification Challenges: (1) Capacity & Conservation Invariants: after every augmentation `f` satisfies `respects_capacity(g, f)` and `is_conserved(g, f, s, t)`; per-vertex conservation needs the `total_flow_in == total_flow_out` axiomatic facts. (2) Termination: Edmonds-Karp's BFS-shortest-path policy ensures the residual shortest-path distance from `s` to `t` is non-decreasing and strictly increases at most `O(VE)` times; provide a loop variant such as `n * E - augmentations_so_far` (or use the integer-capacity argument that the flow value strictly increases each augmentation). (3) Optimality: when BFS finds no augmenting path, `f` is maximum. The classical proof exhibits an `s`-`t` cut whose forward edges are all saturated; in ACSL, you must witness this cut as a ghost set of vertices `S = {v | residual reachable from s}` and prove `flow_val(g, f, s) >= flow_val(g, other, s)` for every other valid flow by reasoning about the cut. ACSL-specific notes: (a) Flow lives in a flat `int64_t f[n*n]` indexed `f[u * (integer)g->n + v]`; the cast `(integer) g->n` is required so the multiplication does not silently overflow `size_t` arithmetic when reasoning. (b) The axiomatic blocks `FlowSums` (sum_flow_out_recursive, sum_flow_in_recursive) define `total_flow_out` and `total_flow_in` keyed by indices into the CSR row of `u` (for out) and over `[0, n)` (for in); proofs require lifting the recursive axioms via inline ghost loops/lemmas, not top-level lemmas. (c) The `is_max_flow` postcondition is universally quantified over `int64_t *other` flow buffers; under the default `Typed` model WP treats this as quantification over phantom buffers, so the optimality argument must be carried by the cut/saturation invariant rather than by a per-`other` SMT search. (d) `assigns f[0 .. (integer)g->n * (integer)g->n - 1]` is the only write. (e) Provide a loop variant for the augmentation outer loop and one for each BFS pass. (f) Default `-wp-model Typed` is sufficient.
