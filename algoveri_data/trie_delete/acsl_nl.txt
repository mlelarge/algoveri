Your task is to implement the Delete operation with Eager Pruning for a Byte-based Trie (Prefix Tree) and verify its functional correctness in ACSL using Frama-C/WP. The trie node is `typedef struct TrieNode { bool is_end; struct TrieNode *children[256]; } TrieNode`. Requirements: Implement `TrieNode *trie_delete(TrieNode *t, const unsigned char *key, size_t len)` that returns the trie with the given key removed. (1) Algorithm: recursively descend along the bytes of key. After the recursive call on `t->children[key[0]]` returns, replace that child pointer with the (possibly NULL) result. At the destination node, clear `is_end`. On the way back up, whenever a node is left with `is_end == false` and all 256 children NULL, free it and return NULL to the caller — this is the "eager pruning" / "trimming" step that keeps the well_formed invariant strict. (2) Functional Correctness: `opt_well_formed(\result)`, `!opt_contains(\result, key, len)`, and frame: `\forall unsigned char *k, integer kl; !seq_eq(k, kl, key, len) ==> (opt_contains(\result, k, kl) <==> opt_contains{Pre}(t, k, kl))`. Verification Challenges: (1) Pruning correctness: after recursion, you must prove that the parent is either still useful (some other child non-null, or is_end true) and therefore well_formed is preserved, or it has become useless and the implementation correctly returns NULL — the well_formed predicate strictly forbids "useless" nodes (`!trie_is_empty`), so any failure to prune breaks the invariant. (2) Structural recursion: well_formed and trie_contains are defined recursively over the heap; the proof obligation at each return point requires reconstructing well_formed of the parent from well_formed of the (possibly modified) children plus the pruning check. (3) Free/allocation reasoning: this is the dominant difficulty. The function may free pruned nodes; WP's allocation/deallocation model has limited automated support, and showing that all freed nodes are disjoint from the still-live tree typically requires explicit ghost bookkeeping. Verus handles this implicitly via Box ownership and drop semantics. (4) Frame property: proving non-touched keys keep their membership requires reasoning that the only structural changes are along the descended path — auxiliary lemmas about subtree disjointness may be needed. ACSL-specific notes: (a) `assigns \nothing;` on the pre-state is acceptable since all writes happen via free / pointer reassignment in fresh return values; document the effective frame as "modifies the tree rooted at t". (b) Use `-wp-model Typed`; recursive node deletion is best implemented iteratively with a stack of parents, but a recursive C implementation matches the recursive predicate structure most naturally. (c) Termination: `loop variant len - i;` for any iteration along the path. (d) Empty key (len == 0): clear `t->is_end`, then prune if useless. (e) No top-level lemmas should be introduced beyond the preamble; the difficulty is concentrated in the allocation-model semantics, which should be flagged as the principal obstacle for WP.
