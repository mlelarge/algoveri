Your task is to implement Merge Sort and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the input is `int32_t *a` of length n with 0 <= n <= 1000 and `n == 0 || \valid(a + (0 .. n - 1))`. Requirements: implement `merge_sort(int32_t *a, size_t n)` so that on return the array is sorted (`is_sorted(a, n)`) and is a permutation of its initial contents (`same_multiset{Pre,Post}(a, n)`). (1) Algorithm: classic divide-and-conquer — recursively sort `a[0 .. mid)` and `a[mid .. n)` (where `mid = n / 2`), then merge the two sorted halves into a temporary buffer and copy back. You may also implement an iterative bottom-up variant. The `merge` step requires either an auxiliary buffer or careful in-place gymnastics; the buffer-based version is dramatically easier to verify. (2) Functional Correctness: `is_sorted(a, n)` follows directly from the merge-step invariant (output is monotone), and `same_multiset{Pre,Post}(a, n)` requires that every recursive call preserves the count_occ of every value across {Pre, Post}. Verification Challenges: (1) Termination of recursion — you must supply a `decreases n` (or per-call ghost ranking) clause; the natural metric is the half-width n. (2) Sortedness of merge — the merge loop invariant must state that the produced prefix is monotone and bounded above by both `a[i]` and `a[j]` (the next unread elements of the two halves). (3) Multiset preservation — this is the dominant difficulty. The count_occ axiomatic preamble defines counts only over a contiguous prefix `a[0..n)`, so when merge writes the result back from the buffer to `a`, you must prove a *three-stage ghost induction*: (i) count_occ of the left half across the recursive call, (ii) count_occ of the right half across the recursive call, (iii) count_occ on the union after the buffer-to-array copy. This is the documented count_occ-preservation-under-right-shift pattern: at the i-th step of the copy-back loop, append a single element to a "written prefix" using count_occ_step, while symmetrically subtracting the same element from a "buffer suffix" using a derived lemma proved by ghost induction inside the function body. ACSL-specific notes: (a) The contract requires `\separated(a + (0 .. n - 1), tmp + (0 .. n - 1))` for the auxiliary buffer (declared as a local VLA or a callee-allocated array). (b) Use `loop variant n - k` for each linear pass; for the recursion, `decreases hi - lo`. (c) Do not introduce free-standing axioms about count_occ — every step must follow from `count_occ_base`, `count_occ_step`, and ghost induction inline. (d) `-wp-model Typed` is sufficient; the int32_t comparisons are handled directly by the integer theory.
