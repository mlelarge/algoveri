Your task is to implement the `query` (range maximum) operation for a Range-Maximum Segment Tree and verify its correctness in ACSL using Frama-C/WP. The tree node is `SegNode { uint64_t val, low, high; SegNode *left, *right; }`. Signature: `uint64_t seg_query(const SegNode *t, uint64_t ql, uint64_t qr)`; the function returns the maximum of `view_at(t, k)` for `k` in the query range `[ql, qr)`. Preconditions: `\valid_read(t)`, `is_segment_tree(t)`, `ql < qr`, and the query range fits inside the node range: `t->low <= ql && qr <= t->high`. The contract specifies `assigns \nothing` and two postconditions stated over the axiomatic logic function `view_at`: (1) Correctness/upper bound — for every `k` in `[ql, qr)`, `view_at(t, k) <= \result`; (2) Tightness/witness — there exists `k` in `[ql, qr)` such that `view_at(t, k) == \result`. The standard segment-tree query proceeds recursively: if the node's range `[t->low, t->high)` is fully contained in `[ql, qr)`, return `t->val` (which by the segment-tree invariant is the maximum across the node's range); else compute `mid = (t->low + t->high) / 2`, recurse into the appropriate child(ren), and return the maximum of the partial answers. Verification challenges: (1) Upper-bound proof — the segment-tree invariant `is_segment_tree` includes the property `t->val = max(t->left->val, t->right->val)`, but proving the universally-quantified upper bound over the entire range requires either an additional invariant `\forall k. in_view(t, k) ==> view_at(t, k) <= t->val` (not directly provided in the preamble) or a recursive argument descending into children. You will likely need a ghost lemma `lemma_val_is_upper_bound(t)` that establishes this from the `is_segment_tree` recursion. (2) Tightness proof — the existential witness must be constructed: when the node range is fully inside `[ql, qr)`, the witness is some leaf attaining `t->val`; in the partial-overlap case it is the witness of whichever child returned the larger value. This requires existential introduction with explicit witnesses passed through ghost code. (3) Termination of the recursion — measure is `t->high - t->low`, halved on each recursive call; standard `loop variant` style applied to the recursive function. ACSL-specific notes: (a) The preamble lacks a "val-is-upper-bound" lemma; you must prove it as a ghost helper or inline in invariants. (b) Use default `-wp-model Typed`; tree is read-only (`assigns \nothing`), no `\separated` needed. (c) The three view axioms suffice for descent, but the existential witness needs a ghost lemma producing a leaf depth realizing the max. (d) No top-level lemmas may be added; local lemma functions discharged by structural induction over `t->high - t->low` are recommended. (e) This problem is SMT-hard at the existential witness postcondition; the contract has no translation defect.
