Your task is to implement the Euclidean Algorithm to compute the Greatest Common Divisor (GCD) of two integers and verify its correctness in ACSL using Frama-C/WP. Preconditions: The inputs a and b are uint64_t. Requirements: Implement the function compute_gcd which returns a uint64_t. (1) Algorithm: Use the classic Euclidean algorithm: repeatedly replace (a, b) with (b, a % b) until b becomes 0. The result is a. (2) Functional Correctness: You must verify that your implementation returns exactly the value defined by the specification spec_gcd, defined in the preamble as an axiomatic logic function with a base case (spec_gcd(a, 0) == a) and step case (spec_gcd(a, b) == spec_gcd(b, a % b) for b > 0). Verification Challenges: (1) Termination: This is the primary challenge. You must prove that the loop terminates by supplying a `loop variant` that strictly decreases. The natural choice is `b` itself, which decreases because a % b < b for b > 0 — Frama-C/WP's integer theory will discharge this automatically. (2) Correctness Invariant: You must verify that the GCD is preserved throughout the transformation. The invariant `spec_gcd(current_a, current_b) == spec_gcd(\at(a, Pre), \at(b, Pre))` must hold at every iteration. Both Alt-Ergo and Z3 can discharge this from the axiomatic step axiom once the variant is in place. ACSL-specific notes: (a) The integer division and modulo (`/`, `%`) on uint64_t inside the function body match the axiomatic spec_gcd's mathematical `%` because both operands are non-negative. (b) The contract uses `assigns \nothing;` since the function is pure. (c) No top-level lemmas are required; the axiomatic GcdModel block in the preamble is the only piece of logic state needed.
