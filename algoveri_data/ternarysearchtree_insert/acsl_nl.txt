Your task is to implement the Insert operation for a Ternary Search Tree (TST) and verify its functional correctness in ACSL using Frama-C/WP. The TST node is `typedef struct TstNode { unsigned char val; bool is_end; struct TstNode *left, *mid, *right; } TstNode`, mirroring the Verus `Node { val, is_end, left/mid/right }`. A TST is a space-efficient hybrid of a Trie and a BST: `left`/`right` form a BST on sibling characters at the current position, while `mid` is the Trie link to the next character. Requirements: Implement `TstNode *tst_insert(TstNode *t, const unsigned char *key, size_t len)` with `len > 0`. (1) Algorithm: if t == NULL, allocate a fresh chain of mid-linked nodes spelling key, with is_end on the last; otherwise compare key[0] to t->val: when smaller recurse into left, when larger into right (key unchanged), when equal and len == 1 set is_end := true, when equal and len > 1 recurse into mid with key+1, len-1. (2) Functional Correctness: the result is non-null, well_formed, not tst_is_empty, satisfies tst_contains(\result, key, len), and the frame `\forall unsigned char *k, integer kl; !seq_eq(k, kl, key, len) ==> (tst_contains(\result, k, kl) <==> opt_contains{Pre}(t, k, kl))`. Verification Challenges: (1) BST invariant preservation: insertion into left/right preserves the strict character ordering because the descent only happens when key[0] < t->val (resp. >); the freshly-allocated chain's root carries that exact byte. (2) Recursive structural induction: well_formed must be reconstructed at every level on the way back up; this is the BST insertion proof with an extra mid axis. (3) Heap allocation: the contract uses `allocates \result;`. WP's allocation model has limited support for `\fresh` and disjointness reasoning between newly allocated nodes and the existing tree typically requires an explicit ghost set of allocated nodes. This is the principal practical obstacle. (4) Frame for non-target keys: only the path from root to the inserted key is structurally modified; proving every other key's membership is preserved requires unfolding tst_contains along disjoint subtrees. ACSL-specific notes: (a) `assigns \nothing;` is acceptable when the implementation always returns a fresh root; if existing nodes are mutated in place, change the contract to `assigns t->is_end, ...;` accordingly. (b) Use `-wp-model Typed`; allocation reasoning is the dominant difficulty. (c) Termination: `loop variant len - i;` for byte-axis descent and a structural variant for left/right. (d) No top-level lemmas should be added — the recursive predicates (well_formed, tst_contains, tst_is_empty) and the BST invariant clause `tst_contains(left, s, sl) ==> sl > 0 && s[0] < t->val` from the preamble are sufficient when loop invariants quote them.
