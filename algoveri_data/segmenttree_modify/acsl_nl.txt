Your task is to implement the functional (path-copying) `modify` operation for a Range-Maximum Segment Tree and verify its correctness in ACSL using Frama-C/WP. The tree node is `SegNode { uint64_t val, low, high; SegNode *left, *right; }`. Signature: `SegNode *seg_modify(SegNode *t, uint64_t idx, uint64_t v)`; given an existing valid segment tree `t`, return a new tree in which `view_at(_, idx) == v` and all other index values are unchanged. Preconditions: `\valid(t)`, `is_segment_tree(t)`, and `t->low <= idx < t->high`. The contract specifies `assigns \nothing`, `allocates \result`, `\result != \null`, `is_segment_tree(\result)`, that the new root's `low/high` equal the original's pre-state values (`\at(t->low, Pre)` and `\at(t->high, Pre)`), and the key view postcondition: for every `k` in `[t->low, t->high)`, `\result` contains `k` in its view and `view_at(\result, k) == (k == idx ? v : view_at{Pre}(t, k))`. The implementation pattern is path-copying: at a leaf representing `idx`, return a new leaf with value `v`; otherwise compute `mid = (t->low + t->high) / 2`, recurse into the child containing `idx`, reuse the pointer to the unchanged sibling, and return a fresh internal node whose `val` is the max of the (new) children. Verification challenges: (1) Termination — the recursion descends one level per call; measure is `t->high - t->low`, strictly halved. (2) Structural invariant — the new internal node must satisfy `is_segment_tree`: child range alignment, the max property `val = max(left->val, right->val)`, and recursive validity (the unchanged child still satisfies `is_segment_tree` because we kept its pointer; the new child does by induction). (3) Frame postcondition — for `k != idx`, the new tree's `view_at` agrees with the old tree's pre-state `view_at`; the descend axioms handle the recursive case but the unchanged sibling needs the equality `view_at(\result->left/right, k) == view_at{Pre}(t->left/right, k)` which holds because the pointer is identical and the underlying memory is not modified (we use `assigns \nothing`). (4) Update postcondition — `view_at(\result, idx) == v` follows by induction by descending into the modified child until the leaf base case. ACSL-specific notes: (a) `assigns \nothing` is essential and verifiable: path copying allocates new nodes (`allocates \result`, transitively) but never writes through the input pointer `t`. (b) Sibling pointer sharing is safe because `is_segment_tree` is read-only. (c) Use default `-wp-model Typed`; freshly allocated nodes are auto-separated. (d) No top-level lemmas may be added; the three view axioms plus structural induction discharge the postconditions. (e) This problem is SMT-hard at the universally-quantified frame postcondition; expect to add ghost lemmas about `view_at` for shared subtrees. The contract has no translation defect.
