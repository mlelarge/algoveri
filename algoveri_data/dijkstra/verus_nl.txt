Your task is to implement Dijkstra's Algorithm for single-source shortest paths and verify its functional correctness in Verus. The input is a WeightedGraph where edge weights are i64. Preconditions: You can assume the graph contains at most 100,000 nodes and edge weights are between -100,000 and 100,000 (inclusive) to prevent integer overflow. Crucially, you can assume all edge weights are non-negative (1$w \ge 0$). Requirements: Implement the function dijkstra_shortest_paths which returns a Vec<Option<i64>>. If a node v is reachable, the vector at index v must contain Some(d), where d is the minimal path weight from the start node to v. If a node v is unreachable, the vector must contain None. Verification Challenges: (1) Relaxation Invariant: Proving that when a node u is extracted from the priority queue, its recorded distance is the true shortest path distance. This relies on the "greedy choice property. (2) "Queue Invariants: Proving that the priority queue correctly maintains the "frontier" of the search. You must prove that for every node in the queue, its key represents the weight of a valid path found so far. If priority queue is hard to implement and prove, you can switch to a general sorted queue (with worse time complexity), but you still need to make sure algorithm (greedy + queue) is correctly implemented and verified. (3) Path Minimality: Proving that no path exists with weight strictly less than the returned distance. The non-negative weight constraint is essential here: it ensures that extending a path never decreases its total weight.
