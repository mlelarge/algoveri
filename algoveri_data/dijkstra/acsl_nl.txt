Your task is to implement Dijkstra's single-source shortest-paths algorithm on a non-negatively weighted directed simple graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is given as a CSR-encoded `WeightedGraph` with parallel `col_idx`/`col_weight` arrays. `well_formed(g)` enforces row-pointer monotonicity, in-range neighbours, and the simple-graph property (no two outgoing edges from `u` share a target). `weights_non_negative(g)` and `weights_and_size_bounded(g)` (n <= 100000, |w| <= 100000) guarantee Dijkstra's correctness and bound arithmetic. The caller provides `int64_t *dist` and `int *reachable` of length `g->n`, and `0 <= start < g->n`. Requirements: Implement `void dijkstra_shortest_paths(const WeightedGraph *g, size_t start, int64_t *dist, int *reachable)` so that for every `v` in `[0, g->n)`: `reachable[v] == 1` and `is_shortest_dist(g, start, v, dist[v])` if `v` is reachable from `start`, otherwise `reachable[v] == 0` and `unreachable_from(g, start, v)`. Use a min-priority queue (binary heap) keyed by tentative distance; on extraction of `u` with finalised distance `d`, relax every edge `(u -> v, w)` by setting `dist[v] = min(dist[v], d + w)`. Verification Challenges: (1) Relaxation Invariant: when `u` is extracted, `dist[u]` is the true shortest-path weight from `start` to `u`. (2) Path Witness: `is_shortest_dist` is an existential over a concrete predecessor path; carry a ghost `parent[]` array so you can reconstruct a witness path of weight `dist[v]`. (3) Lower Bound: for every other path with same endpoints the weight is `>= dist[v]`; this follows from the heap invariant plus non-negativity of weights, but requires lifting the inductive `path_weight` axioms over arbitrary candidate paths. (4) Unreachability: if `reachable[v]` ends `0`, every path is excluded — the loop invariant must say "any path from start whose final vertex is not yet finalised must pass through the heap". ACSL-specific notes: (a) The contract `assigns dist[0 .. g->n - 1], reachable[0 .. g->n - 1];` covers both output buffers, and the postcondition reads them both; without a `\separated(dist + (0 .. g->n - 1), reachable + (0 .. g->n - 1))` clause the WP cannot rule out aliasing, and the `reachable[v] == 1 ==> is_shortest_dist(...)` and `reachable[v] == 0 ==> unreachable_from(...)` clauses become unprovable. This is the precise place where Verus's `&mut` borrow gives the disjointness implicitly. (b) The axiomatic block `DijkstraPathWeight` defines `edge_weight` and `path_weight` recursively keyed on `plen`; drive proofs by induction on path length using inline ghost lemma applications, never via top-level lemmas. (c) The simple-graph property is essential: it makes `edge_weight(u, v)` well-defined (single edge from `u` to `v`). (d) Use the default `Typed` memory model; carry loop variants on heap-extract loop (`n - finalised_count`) and on edge-relaxation loops (`row_ptr[u + 1] - e`). (e) Use `int64_t` arithmetic in the body; with `n <= 100000` and `|w| <= 100000`, path weights fit easily in `i64`.
