Your task is to implement a Naive Discrete Logarithm solver and verify its correctness in Lean. Given a base g, a target h, and a modulus p, find the smallest non-negative integer x < p such that g^x ≡ h (mod p), or return None if no such x exists. Preconditions: p > 1. Postconditions: If result is Some(x), then g^x ≡ h (mod p) and 0 ≤ x < p. Also, there is no k < x such that g^k ≡ h (mod p). If result is None, then no k < p satisfies the condition. Implementation: Iterate from 0 to p-1. Optimization: You are allowed to use standard modular exponentiation for verification, but the naive algorithm simply iterates and multiplies.
