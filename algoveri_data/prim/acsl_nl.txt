Your task is to implement Prim's Algorithm for the Minimum Spanning Tree of a connected, undirected, weighted, simple graph and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the input is a CSR-encoded `WeightedGraph *g` with n nodes (n <= 100,000), `well_formed(g)` (valid row_ptr/col_idx, simple graph), `weights_bounded` (|weight| <= 100,000), and `graph_is_connected(g)`. Output buffers `out_u`, `out_v` (int) and `out_w` (int64_t) each have capacity n - 1. Requirements: implement `prim_mst(g, out_u, out_v, out_w)` returning g->n - 1 and filling the output arrays with the edges of an MST. (1) Algorithm: maintain a `key[v]` (cheapest weight connecting v to the current tree, initially +infinity except key[start] = 0) and a `parent[v]`. Repeatedly extract the unvisited node u with smallest key, mark it visited, append (parent[u], u, key[u]) to the output edges, and relax key[v] for every unvisited neighbor v of u. The standard implementation uses a binary heap or a flat O(N^2) scan. (2) Functional Correctness: prove `\result == g->n - 1` and `is_mst(g, out_u, out_v, out_w, \result)`, which expands to `is_spanning_tree` (every edge in g, every pair of nodes connected via the chosen edges, exactly n - 1 edges) plus *global optimality*: total_weight of the produced edge set is at most that of any other spanning tree. Verification Challenges: (1) Cut Property — every edge added by Prim crosses the cut between visited and unvisited and has minimum weight across that cut. The proof must encode an "exchange argument": for any other spanning tree T', swap a tree edge crossing the cut with the Prim-chosen edge and observe total weight does not increase. (2) Tree Growth Invariant — at every step the chosen edges form a tree (acyclic, connected) that is a subset of *some* MST. Both the connectedness and the "subset of an MST" parts need ghost arrays. (3) Optimality of `is_mst` is research-grade in ACSL — there is no off-the-shelf cut-property lemma library, and the universally-quantified `is_spanning_tree(g, ou, ov, ow, olen) ==> total_weight(...) <= total_weight(...)` requires either a fully manual exchange-argument proof or an axiomatized cut lemma kept inside the function body as ghost code. Note this is the dominant cost of the verification; it is reasonable to leave optimality as an explicit `assert`/`assume` interface and verify only spanning-tree correctness, since a complete proof exceeds typical Frama-C/WP capability. ACSL-specific notes: (a) The contract requires `\separated(out_u + (0 .. n-1), out_v + (0 .. n-1), out_w + (0 .. n-1), g, g->row_ptr + (0 .. n), g->col_idx + (0 .. row_ptr[n] - 1), g->col_weight + (0 .. row_ptr[n] - 1))`; without these the writes to the output arrays cannot be framed against the predicates that read g's internals. (b) Use `loop variant n - visited_count` for the main outer loop. (c) The `total_weight` axiomatic preamble is the only logic state — do not add free-standing axioms; the cut-property reasoning belongs inline as ghost code. (d) `-wp-model Typed` is sufficient; expect Alt-Ergo to discharge spanning-tree obligations and the optimality clause to require manual proof effort.
