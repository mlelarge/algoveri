Your task is to implement a Binary Search Tree membership test and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The input is a (possibly null) pointer `t` to a `BstNode` (a struct `{ uint64_t val; BstNode *left; BstNode *right; }`) satisfying `opt_is_bst(t)` (i.e., either `t == \null`, or `t` is a valid BST: every value in the left subtree is less than `t->val`, every value in the right subtree is greater). The query is a `uint64_t v`. Requirements: Implement `bst_search(const BstNode *t, uint64_t v)` returning `bool` (`true` iff `v` appears in the tree). (1) Algorithm: While `t != NULL`, compare `v` with `t->val`: if equal return `true`; if less, recurse/iterate into `t->left`; otherwise into `t->right`. Return `false` on `NULL`. (2) Functional Correctness: The single postcondition is `(\result != 0) <==> (t != \null && in_view(t, v))`. The preamble defines `in_view{L}(t, x)` recursively: `\valid_read(t) && (t->val == x || (t->left != \null && in_view(t->left, x)) || (t->right != \null && in_view(t->right, x)))`, and `is_bst{L}(t)` likewise recursively, plus `opt_is_bst(t) = (t == \null || is_bst(t))`. Verification Challenges: (1) BST cut-off correctness: the key insight is that descending only one branch is sound because the BST invariant guarantees the other subtree contains no occurrences of `v`. Concretely, if `v < t->val` then no element of the right subtree can equal `v` (since they are all `> t->val > v`); the symmetric argument handles `v > t->val`. (2) Recursion termination: each recursive call decreases the tree's depth/structural size. ACSL-specific notes: (a) `assigns \nothing;` because the function is read-only. (b) `in_view` and `is_bst` are recursive predicates (not in axiomatic blocks); ACSL allows recursion over a recursive data type via the implicit structural decrease on `t`. WP will require a `decreases` clause on each recursive call inside the function body — supply it via a ghost depth or explicitly as `decreases t;` on each predicate use site (or simpler: use iteration instead of recursion). (c) For an iterative implementation, supply `loop invariant in_view{Pre}(\at(t, Pre), v) <==> (t != \null && in_view(t, v));` plus a `loop variant` measuring the height of the remaining subtree (you may need a ghost height function or simply ghost-track the maximum depth). (d) The default `-wp-model Typed` memory model is sufficient — there is no aliasing concern because the function only reads. (e) The `\valid_read(t)` clause inside `is_bst`/`in_view` ensures dereferences are safe; this propagates through recursion as long as `is_bst` was assumed at entry. (f) Per project rules, no top-level lemmas or axioms — encode the cut-off lemma inline as `assert (\forall integer y; in_view(t->right, y) ==> y > t->val);` immediately after binding `t = t->right` etc.
