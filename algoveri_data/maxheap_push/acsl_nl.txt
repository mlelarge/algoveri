Your task is to implement the `maxheap_push` operation for an array-backed binary max-heap and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the heap storage is the prefix `a[0 .. len - 1]` of a uint32_t array of physical capacity `capacity`, with 0 <= len < 1023 and len < capacity, satisfying `is_heap(a, len)`. Requirements: implement `maxheap_push(uint32_t *a, int len, int capacity, uint32_t v)` so that after the call the new heap is `a[0 .. len]` containing the original elements plus the value v, with the heap invariant restored. (1) Algorithm: place v at index `len` (the next free slot), then "sift up" — while the inserted element exceeds its parent at index `(i - 1) / 2`, swap them and move i up. (2) Functional Correctness: prove `is_heap(a, len + 1)` holds after sift-up and `view_inserted{Pre,Post}(a, len, v)` — namely `\forall x. count_occ{Post}(a, len + 1, x) == count_occ{Pre}(a, len, x) + (x == v ? 1 : 0)`. Verification Challenges: (1) Multiset preservation under sift-up — every swap exchanges two cells, and the global count_occ predicate must be threaded through the loop using the same ghost-loop pattern documented for `bubble_sort`: at every iteration the running invariant is `\forall x. count_occ{Here}(a, len + 1, x) == count_occ{Pre}(a, len, x) + (x == v ? 1 : 0)`, re-established symmetrically by appealing to `count_occ_step` on each modified suffix. (2) Heap invariant — the standard sift-up loop invariant is "the heap property holds for all indices except possibly the path from the inserted node to the root, and at every node off that path the original parent-child relation is preserved." (3) Capacity bound — len + 1 must remain <= 1023 to satisfy the new is_heap precondition; this comes from `len < 1023`. ACSL-specific notes: (a) The contract requires `\separated` to be added: `\separated(a + (0 .. capacity - 1), \result)` is moot here (void return), but you should note that the `assigns a[0 .. len]` clause depends on the cells being disjoint from any caller-side aliases — if v itself were derived from a[i] (it is a plain uint32_t value here, so it is fine). (b) Use `loop variant i` for the sift-up (i decreases towards 0). (c) The `count_occ` axiomatic preamble plus the derived `view_inserted` predicate are the only logic state — do not introduce free-standing lemmas; multiset arithmetic should be carried out as ghost steps inside the loop. (d) `-wp-model Typed` is sufficient.
