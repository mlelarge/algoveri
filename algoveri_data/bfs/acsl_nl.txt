Your task is to implement a Breadth-First Search (BFS) shortest-path algorithm and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: The graph is a CSR-encoded `Graph` (a struct with `n`, `row_ptr` of length `n+1`, and `col_idx` of length `row_ptr[n]`) satisfying `well_formed(g)` and `size_bounded(g)` (`g->n <= 10000`). The `start` and `target` indices are within `[0, g->n)`. Requirements: Implement `bfs_shortest_path` returning an `int`: a non-negative value is the BFS distance (in number of edges) from `start` to `target`; the sentinel value `-1` indicates `target` is not reachable from `start`. (1) Algorithm: Use a FIFO queue and a `visited[]`/`distance[]` table. Push `start` with distance `0`, and on each dequeue, push every unvisited neighbor with `distance+1`. Stop when `target` is dequeued (or the queue empties). (2) Functional Correctness: The postconditions are `\result >= -1`, `\result == -1 <==> !reachable(g, start, target)`, and `\result >= 0 ==> is_shortest_distance(g, start, target, \result)`, where `is_shortest_distance(g, start, target, d)` says some path of edge-length `d` exists and every path is at least that long. `reachable(g, s, t)` is defined as `\exists int *p, integer plen; is_path(g, p, plen) && p[0] == s && p[plen-1] == t`. Verification Challenges: (1) Minimality: it is not enough to find any path; you must prove no path with fewer edges exists. (2) Queue layer invariant: at every point the queue contains only nodes at distance `k` followed by nodes at distance `k+1`, and they are processed level-by-level. (3) Reachability proof for the negative result: if the queue empties without seeing `target`, every node reachable from `start` has been visited and `target` wasn't among them, so `!reachable(g, start, target)`. ACSL-specific notes: (a) The contract is `assigns \nothing;` — internally you may use stack/local arrays of size up to `g->n` for the visited/distance/queue buffers (these do not appear in `assigns` because they are local). (b) The preamble's predicates `is_path` and `reachable` quantify existentially over `int *p` — this is the standard ACSL idiom for "exists a sequence", and Frama-C/WP treats `p` as an arbitrary readable buffer; your invariants should construct/exhibit such buffers using ghost arrays where necessary. (c) `is_shortest_distance` involves both an existential (a witness path of length `d+1`) and a universal (every path is at least that long). The universal part typically requires a ghost loop that maintains "for every node `v` already dequeued, the distance recorded is the shortest". (d) The default `-wp-model Typed` memory model is sufficient since the graph is read-only and all writes are to local buffers. (e) Supply `loop variant` clauses for both the outer "queue not empty" loop (the variant is `g->n - dequeue_count`) and any inner "scan neighbors" loop (variant `row_ptr[u+1] - e`). (f) Per project rules, no top-level lemmas or axioms in the user solution; encode all auxiliary facts as `assert` and `loop invariant`.
