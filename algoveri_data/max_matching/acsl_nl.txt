Your task is to implement an algorithm for Maximum Bipartite Matching and verify its functional correctness in ACSL using Frama-C/WP. Preconditions: the input is a bipartite graph encoded as a CSR struct `BipartiteGraph` with `left_size` and `right_size` (each at most 1000), a `row_ptr` array of length `left_size + 1`, and a `col_idx` array of length `row_ptr[left_size]`. Each entry of `col_idx[e]` for e in [row_ptr[u], row_ptr[u+1]) is a right-side neighbor in [0, right_size). Requirements: implement `max_bipartite_matching` which returns a size_t equal to the size of a maximum matching. (1) Algorithm: Kuhn's DFS-based augmenting-path algorithm — maintain a `match_v` array indexed by right-side nodes; for every left node u in turn, run a DFS through u's neighbors v, recursively reassigning match_v[v] if its current owner can be re-matched along an augmenting path; increment the result whenever a new augmenting path is found. (2) Soundness: the contract states `\exists int *mu, int *mv; is_max_matching(g, mu, mv, \result)`, so you must establish, via ghost arrays, that the parallel-array matching `(mu[0..\result), mv[0..\result))` actually exists, every recorded edge is in the adjacency list (`matching_valid_edges`), and no two edges share an endpoint (`is_disjoint`). (3) Optimality: the predicate `is_max_matching` quantifies over every other parallel-array matching of any length, requiring you to invoke Berge's lemma — a matching is maximum iff no augmenting path remains in the residual graph. Verification Challenges: (1) Ghost reconstruction — the C signature returns only a count, but the contract demands existence of a witness, so you must populate ghost `mu`, `mv` arrays during the DFS. (2) Berge's lemma — proving optimality from "the DFS returned no path for u" is research-grade in any proof assistant; in ACSL it typically requires a hand-written invariant on the visited set plus an inductive argument over augmenting paths, encoded as ghost code. (3) Termination of the recursive DFS — needs a `loop variant` (or, if recursive, a `decreases` clause) bounded by `right_size - |visited|`. ACSL-specific notes: (a) The contract requires `\separated(g, g->row_ptr + (0 .. left_size), g->col_idx + (0 .. row_ptr[left_size] - 1))` so the WP frame stays trivial under `assigns \nothing`. (b) The optimality clause is the dominant difficulty; expect to need an axiomatic block that defines "augmenting path" and a top-level-free ghost lemma chain inside the function body — do not introduce free-standing axioms. (c) Use `-wp-model Typed`; no manual triggers are needed because the existential is closed by a single ghost witness. (d) Bound recursion depth via the size_bounded predicate (left_size <= 1000) so termination metrics stay within int.
